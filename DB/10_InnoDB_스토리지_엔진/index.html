10. InnoDB мҠӨнҶ лҰ¬м§Җ м—”м§„ -- DBмқҳ мӢ¬мһҘл¶Җ¶

1. InnoDBлһҖ?¶

1.1 н•ң л¬ёмһҘ м •мқҳ¶

"MariaDB/MySQLмқҳ кё°ліё мҠӨнҶ лҰ¬м§Җ м—”м§„. нҠёлһңмһӯм…ҳ, н–ү л ҲлІЁ лқҪ, мһҘм•  ліөкө¬лҘј м§Җмӣҗн•ңлӢӨ."

1.2 мҠӨнҶ лҰ¬м§Җ м—”м§„мқҙ лӯ”к°Җ¶

MySQL/MariaDBмқҳ лҸ…нҠ№н•ң кө¬мЎ°

мҠӨнҶ лҰ¬м§Җ м—”м§„мқҙлһҖ?

мҠӨнҶ лҰ¬м§Җ м—”м§„ = лҚ°мқҙн„°лҘј м–ҙл–»кІҢ м ҖмһҘн•ҳкі  мқҪмқ„м§Җ кІ°м •н•ҳлҠ” л¶Җл¶„. InnoDB = нҳ„мһ¬ кё°ліёмқҙмһҗ мӮ¬мӢӨмғҒ мң мқјн•ң м„ нғқм§Җ.

  • мғҲлЎң мқҪмқҖ нҺҳмқҙм§ҖлҠ” Old мҳҒм—ӯм—җ лЁјм Җ л“Өм–ҙк°„лӢӨ

4.2 Dirty Page¶

Dirty Page

Dirty Page = л©”лӘЁлҰ¬м—җм„ң мҲҳм •лҗҗм§Җл§Ң л””мҠӨнҒ¬м—җ м•„м§Ғ м•Ҳ м“ҙ нҺҳмқҙм§Җ

UPDATE tb_user SET name = 'к№ҖмІ мҲҳ' WHERE id = 1;

  • Step 1: Buffer Poolм—җм„ң н•ҙлӢ№ нҺҳмқҙм§ҖлҘј м°ҫлҠ”лӢӨ
  • Step 2: л©”лӘЁлҰ¬мқҳ нҺҳмқҙм§ҖлҘј мҲҳм •н•ңлӢӨ
  • Step 3: Redo лЎңк·ём—җ ліҖкІҪ лӮҙмҡ©мқ„ кё°лЎқн•ңлӢӨ
  • Step 4: л””мҠӨнҒ¬мқҳ .ibd нҢҢмқјмқҖ м•„м§Ғ м•Ҳ кі м№ңлӢӨ! (Dirty)

мҷң л””мҠӨнҒ¬м—җ л°”лЎң м•Ҳ м“°лӮҳ?

  • л””мҠӨнҒ¬ м“°кё°лҠ” лҠҗлҰ¬лӢӨ
  • л§Ө UPDATEл§ҲлӢӨ л””мҠӨнҒ¬м—җ м“°л©ҙ м„ұлҠҘ н•ҳлқҪ
  • л©”лӘЁлҰ¬м—җ лӘЁм•„л’ҖлӢӨк°Җ лӮҳмӨ‘м—җ н•ң лІҲм—җ м“ҙлӢӨ

  • Redo лЎңк·ёк°Җ мһҲлӢӨ (Step 3м—җм„ң кё°лЎқн•Ё)
  • мһҘм•  ліөкө¬ мӢң Redo лЎңк·ёлҘј мһ¬мғқн•ҳл©ҙ лҗңлӢӨ
  • мқҙкІғмқҙ WAL (Write-Ahead Logging)

Checkpoint:

Checkpoint = Dirty PageлҘј л””мҠӨнҒ¬м—җ м“°лҠ” мӢңм җ

InnoDBлҠ” мЈјкё°м ҒмңјлЎң Dirty PageлҘј л””мҠӨнҒ¬(.ibd)м—җ кё°лЎқн•ңлӢӨ. мқҙкІғмқ„ Checkpointлқјкі  н•ңлӢӨ. Checkpointк№Ңм§Җ мҷ„лЈҢлҗң лҚ°мқҙн„°лҠ” Redo лЎңк·ё м—ҶмқҙлҸ„ м•Ҳм „н•ҳлӢӨ. Checkpoint мқҙнӣ„мқҳ ліҖкІҪл§Ң Redo лЎңк·ё мһ¬мғқмқҙ н•„мҡ”н•ҳлӢӨ.

Checkpointк°Җ мӨ‘мҡ”н•ң мқҙмң : Redo лЎңк·ё нҢҢмқјмқҖ нҒ¬кё°к°Җ кі м •мқҙлӢӨ (мҲңнҷҳ нҢҢмқј). Checkpoint мқҙм „мқҳ Redo лЎңк·ёлҠ” лҚ®м–ҙм“ё мҲҳ мһҲлӢӨ. Checkpointк°Җ л°ҖлҰ¬л©ҙ Redo лЎңк·ёк°Җ к°Җл“қ м°Ём„ң м“°кё°к°Җ л©Ҳм¶ҳлӢӨ.

мқҙкІғмқҖ DBмқҳ н•өмӢ¬ мӣҗлҰ¬ мӨ‘ н•ҳлӮҳлӢӨ. мҷң лҚ°мқҙн„°лҘј л°”лЎң л””мҠӨнҒ¬м—җ м•Ҳ м“°кі  лЎңк·ёлҘј лЁјм Җ м“°лӮҳ?

м“°кё° л°©мӢқ I/O нҢЁн„ҙ мҶҚлҸ„

м „лһө: ліҖкІҪ лӮҙмҡ©мқ„ Redo лЎңк·ём—җ лЁјм Җ мҲңм°Ём ҒмңјлЎң м“ҙлӢӨ (л№ лҰ„). мӢӨм ң лҚ°мқҙн„° нҺҳмқҙм§ҖлҠ” лӮҳмӨ‘м—җ н•ң лІҲм—җ м“ҙлӢӨ (Checkpoint). мһҘм•  лӮҳл©ҙ? Redo лЎңк·ё мһ¬мғқмңјлЎң ліөкө¬.

6.1 ROLLBACKмҡ©¶

Undo лЎңк·ём—җ кё°лЎқ: "id=1мқҳ nameмқҖ мӣҗлһҳ 'л°•мҳҒнқ¬'мҳҖлӢӨ"

ROLLBACK мӢӨн–ү мӢң: Undo лЎңк·ёлҘј мқҪлҠ”лӢӨ вҶ’ nameмқ„ 'л°•мҳҒнқ¬'лЎң ліөмӣҗн•ңлӢӨ вҶ’ ліҖкІҪ м·ЁмҶҢ мҷ„лЈҢ

6.2 MVCCмҡ© (лӢӨмӨ‘ лІ„м „ лҸҷмӢңм„ұ м ңм–ҙ)¶

MVCC = Multi-Version Concurrency Control

м—¬лҹ¬ нҠёлһңмһӯм…ҳмқҙ лҸҷмӢңм—җ к°ҷмқҖ лҚ°мқҙн„°лҘј мқҪм–ҙлҸ„ м¶©лҸҢ м•Ҳ лӮҳкІҢ н•ҳлҠ” кё°мҲ .

  • нҠёлһңмһӯм…ҳ A: UPDATE name = 'к№ҖмІ мҲҳ' (м•„м§Ғ COMMIT м•Ҳ н•Ё)
  • нҠёлһңмһӯм…ҳ B: SELECT name ... (лӯҗк°Җ ліҙм—¬м•ј н•ҳлӮҳ?)

кІ©лҰ¬ мҲҳмӨҖм—җ л”°лқј READ COMMITTEDм—җм„ңлҠ” нҠёлһңмһӯм…ҳ Aмқҳ COMMIT м „ лІ„м „ = 'л°•мҳҒнқ¬'к°Җ ліҙмқёлӢӨ. м–ҙл””м„ң? Undo лЎңк·ём—җм„ң!

Undo лЎңк·ём—җ мқҙм „ лІ„м „мқҙ лӮЁм•„ мһҲмңјлҜҖлЎң нҠёлһңмһӯм…ҳ BлҠ” Undo лЎңк·ём—җм„ң мқҙм „ лІ„м „мқ„ мқҪлҠ”лӢӨ. нҠёлһңмһӯм…ҳ Aк°Җ COMMIT н•ҳл©ҙ к·ёл•Ң мөңмӢ  лІ„м „мқҙ ліҙмқёлӢӨ.

мқҙкІғмқҙ "н–ү л ҲлІЁ лқҪ м—ҶмқҙлҸ„ мқҪкё°к°Җ к°ҖлҠҘн•ң" мқҙмң лӢӨ. мқҪкё°лҠ” Undoм—җм„ң мқҙм „ лІ„м „мқ„ мқҪмңјл©ҙ лҗҳлӢҲк№Ң лқҪмқҙ н•„мҡ” м—ҶлӢӨ. 04мһҘм—җм„ң л°°мҡҙ кІ©лҰ¬ мҲҳмӨҖмқҳ л¬јлҰ¬м Ғ кө¬нҳ„мІҙк°Җ л°”лЎң Undo лЎңк·ёлӢӨ.

л¬ём ң 3: л§Өмқј 00:00 л°ұм—… 14к°ңмӣ” мӢӨнҢЁ (0л°”мқҙнҠё)

  • mysqldumpк°Җ 43GB н…Ңмқҙлё”мқ„ лҚӨн”„н•ҳл ӨлӢӨ мӢңк°„ мҙҲкіј
  • лҳҗлҠ” л””мҠӨнҒ¬ кіөк°„ л¶ҖмЎұмңјлЎң лҚӨн”„ нҢҢмқј мғқм„ұ мӢӨнҢЁ
  • л°ұм—… мҠӨнҒ¬лҰҪнҠём—җ м—җлҹ¬ н•ёл“Өл§Ғмқҙ м—Ҷм–ҙм„ң мӢӨнҢЁлҘј 14к°ңмӣ”к°„ лӘЁлҰ„
  • мқҙкІғмқҙ "лӘЁлӢҲн„°л§Ғ м—ҶлҠ” л°ұм—…мқҖ л°ұм—…мқҙ м•„лӢҲлӢӨ"мқҳ мӢӨм ң мӮ¬лЎҖ
-- binlog нҷ•мқё л°Ҹ кҙҖлҰ¬ мҝјлҰ¬

-- нҳ„мһ¬ binlog нҢҢмқј лӘ©лЎқ
SHOW BINARY LOGS;
-- binlog л§ҢлЈҢ м„Өм • нҷ•мқё

-- 7мқј мқҙм „ binlog мӮӯм ң

-- л§ҢлЈҢ кё°к°„ м„Өм • (м„ңлІ„ мһ¬мӢңмһ‘ мӢң мҙҲкё°нҷ”лҗҳлҜҖлЎң my.cnfм—җлҸ„ м¶”к°Җ)
SET GLOBAL expire_logs_days = 7;

8. н•өмӢ¬ м •лҰ¬¶

10мһҘ н•өмӢ¬ м •лҰ¬: InnoDB мҠӨнҶ лҰ¬м§Җ м—”м§„

  1. InnoDB = нҠёлһңмһӯм…ҳ + н–ү л ҲлІЁ лқҪ + мһҘм•  ліөкө¬

    • MariaDB/MySQLмқҳ кё°ліё м—”м§„, мӮ¬мӢӨмғҒ мң мқјн•ң м„ нғқм§Җ

  2. Buffer Pool = DB м„ұлҠҘмқҳ н•өмӢ¬

    • л””мҠӨнҒ¬ лҚ°мқҙн„°лҘј л©”лӘЁлҰ¬м—җ мәҗмӢң
    • LRU(Young/Old)лЎң кҙҖлҰ¬, н’ҖмҠӨмә” мҳӨм—ј л°©м§Җ
    • Dirty Page: л©”лӘЁлҰ¬м—җм„ң мҲҳм •лҗЁ, л””мҠӨнҒ¬м—җ м•„м§Ғ м•Ҳ м”Җ
    • Checkpointм—җм„ң Dirty PageлҘј л””мҠӨнҒ¬м—җ кё°лЎқ

  3. .ibd нҢҢмқј = н…Ңмқҙлё” лҚ°мқҙн„°мқҳ л¬јлҰ¬м Ғ мӢӨмІҙ

    • innodb_file_per_table=ONмқҙл©ҙ н…Ңмқҙлё”лӢ№ 1к°ң
    • 16KB нҺҳмқҙм§Җ лӢЁмң„лЎң кө¬м„ұ
    • TRUNCATE: нҢҢмқј мӮӯм ң + мһ¬мғқм„ұ (43GB вҶ’ 0.02MB, мҲҳ л¶„)
    • DELETE: н–ү л§ҲнӮ№л§Ң, нҢҢмқј нҒ¬кё° м•Ҳ мӨ„м–ҙл“Ұ (43GB вҶ’ 43GB)

  4. Redo лЎңк·ё = WAL (Write-Ahead Logging)

    • лҚ°мқҙн„°ліҙлӢӨ лЎңк·ёлҘј лЁјм Җ м“ҙлӢӨ
    • Sequential I/Oлқјм„ң л№ лҘҙлӢӨ
    • мһҘм•  мӢң лЎңк·ё мһ¬мғқмңјлЎң ліөкө¬
    • мҲңнҷҳ нҢҢмқј кө¬мЎ° (кі м • нҒ¬кё°)

  5. Undo лЎңк·ё = ROLLBACK + MVCCмқҳ кё°л°ҳ

    • ліҖкІҪ м „ лҚ°мқҙн„°лҘј кё°лЎқ
    • ROLLBACK мӢң мқҙм „ мғҒнғңлЎң ліөмӣҗ
    • MVCC: лӢӨлҘё нҠёлһңмһӯм…ҳмқҙ мқҙм „ лІ„м „мқ„ мқҪмқ„ мҲҳ мһҲкІҢ н•Ё
    • лҢҖмҡ©лҹү DELETE мӢң Undoк°Җ нҸӯл°ңн•ңлӢӨ (м„ңлІ„ мӮ¬л§қ мӣҗмқё)

  6. Binary лЎңк·ё = ліөм ң + PITR

    • Redo лЎңк·ёмҷҖ лӢӨлҘҙлӢӨ (м„ңлІ„ л ҲлІЁ, лҲ„м Ғ, л…јлҰ¬м Ғ)
    • кҙҖлҰ¬ м•Ҳ н•ҳл©ҙ л””мҠӨнҒ¬ к°Җл“қ м°¬лӢӨ
    • expire_logs_days м„Өм • н•„мҲҳ

  7. мҡ°лҰ¬ мӮ¬лЎҖмқҳ көҗнӣҲ

    • 43GB .ibd: мҪ”л“ң кІ°н•ЁмңјлЎң лҚ°мқҙн„° нҸӯмҰқ
    • 14к°ңмӣ” л°ұм—… мӢӨнҢЁ: лӘЁлӢҲн„°л§Ғ м—ҶлҠ” л°ұм—…мқҖ л°ұм—…мқҙ м•„лӢҲлӢӨ
    • binlog лҲ„м Ғ: кҙҖлҰ¬ м„Өм • м—Ҷмңјл©ҙ л””мҠӨнҒ¬ мһЎм•„лЁ№лҠ”лӢӨ
    • мў…н•©: мҡҙмҳҒмқҖ л§Ңл“ңлҠ” кІғліҙлӢӨ кҙҖлҰ¬н•ҳлҠ” кІғмқҙ м–ҙл өлӢӨ
