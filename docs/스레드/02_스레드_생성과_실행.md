# 02. 스레드 생성과 실행

**이 장의 목표**: Java에서 스레드를 만드는 방법 2가지를 알고, 차이를 설명할 수 있다

---

## 1. 스레드 만드는 방법 2가지

### 1.1 방법 1: Thread 클래스 상속

```java
// 1. Thread를 상속받는 클래스 만들기
class MyThread extends Thread {
    @Override
    public void run() {
        // 이 안에 스레드가 할 일을 적는다
        System.out.println("스레드 실행 중: " + getName());
    }
}

// 2. 사용
MyThread t = new MyThread();
t.start();  // ← run()이 아니라 start()를 호출해야 한다!
```

### 1.2 방법 2: Runnable 인터페이스 구현

```java
// 1. Runnable을 구현하는 클래스 만들기
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("작업 실행 중");
    }
}

// 2. 사용
Thread t = new Thread(new MyTask());
t.start();
```

### 1.3 익명 클래스 / 람다로 간단하게

```java
// 익명 클래스
Thread t1 = new Thread(new Runnable() {
    public void run() {
        System.out.println("익명 클래스 스레드");
    }
});
t1.start();

// 람다 (Java 8+)
Thread t2 = new Thread(() -> {
    System.out.println("람다 스레드");
});
t2.start();
```

---

## 2. Thread 상속 vs Runnable 구현

!!! tip "Thread 상속 vs Runnable 구현"

    **Thread 상속:** `class MyThread extends Thread { ... }`

    - 장점: 간단함
    - 단점: Java는 단일 상속 -- 다른 클래스 상속 못 함

    **Runnable 구현:** `class MyTask implements Runnable { ... }`

    - 장점: 다른 클래스 상속 가능, 인터페이스니까
    - 단점: Thread 객체를 별도로 만들어야 함

    **결론: Runnable이 더 좋다.** 실무에서는 대부분 Runnable (또는 Callable) 사용

## 3. start() vs run() — 이거 진짜 중요

### 3.1 차이

```java
Thread t = new Thread(() -> {
    System.out.println("현재 스레드: " + Thread.currentThread().getName());
});

t.run();    // ❌ main 스레드에서 그냥 메서드 호출 (새 스레드 안 만듦)
t.start();  // ✅ 새 스레드를 만들고 그 스레드에서 run() 실행
```

### 3.2 뭐가 다른지

```
t.run() 호출:
  main 스레드 → run() 실행 → 다음 코드
  (스레드 1개로 순차 실행, 의미 없음)

t.start() 호출:
  main 스레드 → 새 스레드 생성 → 새 스레드가 run() 실행
  main 스레드 → 바로 다음 코드 실행 (기다리지 않음)
  (스레드 2개가 동시에 돌아감)
```

!!! warning "run() vs start() 핵심"

    - **run():** 일반 메서드 호출. 새 스레드 안 생김.
    - **start():** JVM이 새 스레드를 만들고, 그 스레드에서 run() 실행

    **start() 내부 동작:**

    1. JVM에 새 스레드 생성 요청
    2. OS가 새 스레드 할당
    3. 새 스레드의 스택 메모리 할당
    4. 새 스레드가 run() 실행

    start()를 호출해야 "진짜" 멀티스레드

### 3.3 start()는 한 번만 호출 가능

```java
Thread t = new Thread(() -> System.out.println("실행"));
t.start();   // OK
t.start();   // IllegalThreadStateException! 한 번 시작한 스레드는 재시작 불가
```

## 4. 스레드 이름

### 4.1 기본 이름

```java
Thread t1 = new Thread(() -> {});  // Thread-0
Thread t2 = new Thread(() -> {});  // Thread-1
Thread t3 = new Thread(() -> {});  // Thread-2
// 자동으로 번호 붙음
```

### 4.2 이름 지정

```java
Thread t = new Thread(() -> {}, "내-스레드");
// 또는
t.setName("내-스레드");
```

### 4.3 우리 서버에서 본 이름들

```
Timer-0     ← new Timer(true) 하면 자동으로 붙는 이름
Timer-1     ← 두 번째 Timer
http-nio-8080-exec-1  ← Tomcat 요청 처리 스레드
main        ← 메인 스레드
```

## 5. 우리 프로젝트 코드로 보기

```java
// RefreshableSqlSessionFactoryBean.java

timer = new Timer(true);  // ← 여기서 Timer-0 스레드 생성
```

`new Timer(true)`가 내부적으로 하는 일:

```java
// java.util.Timer 내부 코드 (JDK 소스)
public Timer(boolean isDaemon) {
    this("Timer-" + serialNumber(), isDaemon);
}

public Timer(String name, boolean isDaemon) {
    thread.setName(name);        // 이름: "Timer-0"
    thread.setDaemon(isDaemon);  // 데몬 스레드: true
    thread.start();              // ← 여기서 스레드 시작!
}
```

```
즉:
  new Timer(true)
  → Timer 내부에서 new Thread() + start()
  → Timer-0 스레드 탄생
  → 이 스레드가 TimerTask를 반복 실행
```

## 6. 실행 순서는 보장 안 됨

```java
Thread t1 = new Thread(() -> System.out.println("A"));
Thread t2 = new Thread(() -> System.out.println("B"));
Thread t3 = new Thread(() -> System.out.println("C"));

t1.start();
t2.start();
t3.start();

// 출력 결과: A B C? B A C? C A B?
// → 모른다. OS 스케줄러가 결정한다.
// → 실행할 때마다 달라질 수 있다.
```

!!! danger "실행 순서 비결정성"

    스레드 실행 순서는 프로그래머가 제어 못 한다.
    OS 스케줄러가 "누가 먼저 CPU를 쓸지" 결정한다.

    이게 멀티스레드가 어려운 이유 중 하나다.
    "될 때도 있고 안 될 때도 있는" 버그가 생기는 원인이다.

## 7. 핵심 정리

!!! abstract "핵심 정리"

    **스레드 생성 방법:**

    1. Thread 상속 (단일 상속 제약)
    2. Runnable 구현 (권장)

    **start() vs run():**

    - start() = 새 스레드 만들어서 실행 (진짜 멀티스레드)
    - run() = 현재 스레드에서 그냥 호출 (의미 없음)

    **실행 순서:** OS가 결정, 프로그래머가 제어 못 함

    **우리 서버:** new Timer(true) -- 내부에서 Thread 생성 + start() -- Timer-0 스레드 탄생

    **다음 장:** 스레드의 생명주기 -- NEW, RUNNABLE, WAITING, TERMINATED 등
