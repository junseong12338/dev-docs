## 핵심 원칙

```
1. 스레드를 '안다'는 건 왜 필요한지 설명할 수 있다는 뜻이다
2. 동기화를 '안다'는 건 문제 상황을 예측할 수 있다는 뜻이다
3. 스레드 문제를 '안다'는 건 코드만 보고 위험을 찾을 수 있다는 뜻이다
```

## 우리가 분석할 실제 코드

```java
// RefreshableSqlSessionFactoryBean.java (메모리 누수 원인)
timer = new Timer(true);        // Timer-0 스레드 생성
timer.schedule(task, 0, 1000);  // 1초마다 task 실행

// MAT 분석 결과
Timer-0 → TimerTask → RefreshableSqlSessionFactoryBean → DelegatingClassLoader 4,257개
```

이 코드가 왜 문제인지, 스레드의 어떤 특성 때문인지.
전부 씹어먹는다.
