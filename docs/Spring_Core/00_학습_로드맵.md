# 00. Spring Core 완전정복 - 학습 로드맵

**미션**: Spring이 뭔지, Bean이 뭔지, DI가 뭔지 자다가 일어나서 설명 가능해야 한다

---

## 왜 이걸 배워야 하나

```
우리 프로젝트 context-sqlMap.xml에서 이걸 봤다:

<bean id="sqlSession" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource" />
</bean>

이 한 줄이 뭔지 모르면:
  → Bean이 뭔지 모르는 거
  → DI가 뭔지 모르는 거
  → Spring이 뭔지 모르는 거
  → 즉, 프레임워크 위에서 일하면서 프레임워크를 모르는 거

그건 운전하면서 핸들이 뭔지 모르는 것과 같다.
```

---

## 전체 구조

!!! abstract "Spring Core 완전정복 커리큘럼"

    **[01장] Spring이 뭐야**
    : IoC, 프레임워크 vs 라이브러리, 제어의 역전
    : "내가 호출하는 게 아니라 Spring이 호출한다"

    **[02장] Bean이 뭐야**
    : 객체 등록, Spring 컨테이너, new 안 쓰는 이유
    : "Spring이 대신 만들어주는 객체"

    **[03장] DI - 의존성 주입**
    : 생성자/setter/필드 주입, 왜 직접 new 안 하는지
    : "필요한 걸 Spring이 알아서 넣어준다"

    **[04장] XML 설정 완전정복**
    : bean, property, ref, value, 우리 프로젝트 XML 해석
    : "XML 한 줄 한 줄이 다 Spring한테 내리는 명령"

    **[05장] Bean 생명주기**
    : 생성 → 초기화 → 사용 → 소멸, afterPropertiesSet, destroy
    : "Bean은 태어나고, 일하고, 죽는다"

    **[06장] Bean Scope**
    : singleton vs prototype, 왜 기본이 singleton인지
    : "같은 Bean을 요청하면 같은 놈이 나온다"

    **[07장] 어노테이션 설정**
    : @Component, @Service, @Autowired, @Resource
    : "XML 대신 코드에 직접 표시한다"

    **[08장] Spring Context**
    : ApplicationContext, Root vs Servlet Context
    : "Bean들이 사는 세상 = Context"

    **[09장] 우리 프로젝트로 보는 Spring**
    : context-sqlMap.xml, context-common.xml 실전 분석
    : "이론을 우리 코드에서 확인한다"

    **[10장] 빠싺 최종시험**
    : 답 못하면 해당 챕터 다시 읽기

## 선수 지식

```
필요한 것:
  - Java 기본 문법 (클래스, 인터페이스, 상속)
  - 객체가 뭔지 아는 것
  - new로 객체 만드는 거 해본 것

없어도 되는 것:
  - Spring 경험 (0부터 시작)
  - 디자인 패턴 지식 (필요하면 그때 설명)
  - 웹 개발 경험
```
