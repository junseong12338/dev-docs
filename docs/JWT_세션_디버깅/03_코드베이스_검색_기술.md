# 03. 코드베이스 검색 기술 - "어떻게 찾아야 하는가"

> 👹 "업체가 힌트를 다 줬는데 못 찾은 건 검색 능력이 없어서야.
> 검색은 기술이다. 감으로 하는 게 아니야."

---

## 검색의 3원칙

| 순서 | 원칙 | 설명 |
|------|------|------|
| 1 | 키워드 추출 | 뭘 검색할 건지 |
| 2 | 도구 선택 | 어디서 검색할 건지 |
| 3 | 범위 좁히기 | 결과가 너무 많으면 필터링 |

---

## 원칙 1: 키워드 추출

업체 답변을 다시 보자:

> "CMS Connector에 접속할때 담는 **jwtToken**의 유효시간이 어떻게 되는지"
> "**tokenExpire**를 직접 만들어서"
> "**JwtProvider**를 사용"
> "**Constants.CMS_JWT_API_KEY**"

여기서 검색 키워드가 **4개** 나온다:

| 키워드 | 왜 이걸로 검색? |
|--------|-----------------|
| `tokenExpire` | 변수명 직접 언급됨 |
| `JwtProvider` | 클래스명 직접 언급됨 |
| `CMS_JWT` | 상수명 패턴 |
| `jwtToken` | form data 파라미터명 |

> 👹 **"상대방이 코드에서 쓰이는 이름을 직접 말해줬으면, 그게 검색어야.
> 자연어로 '토큰 만료 설정 어디있지~' 이렇게 찾는 게 아니야."**

---

## 원칙 2: 도구 선택

### IDE 전체 검색 (Ctrl+Shift+F)

가장 기본이고 가장 강력하다.

!!! tip "IDE 전체 검색"
    IntelliJ / Eclipse / VS Code 전부 지원:

    `Ctrl+Shift+F` → "tokenExpire" 입력
    → 프로젝트 전체에서 이 문자열이 나오는 곳 전부 표시

| 도구 | 단축키 | 특징 |
|------|--------|------|
| IntelliJ | Ctrl+Shift+F | 정규식 지원, 파일 타입 필터 |
| Eclipse | Ctrl+H → File Search | 프로젝트/워크스페이스 범위 선택 |
| VS Code | Ctrl+Shift+F | 빠름, glob 패턴 지원 |

### 터미널 검색 (grep / rg)

IDE 없이 서버에서 직접 찾아야 할 때:

```bash
# grep (기본)
grep -rn "tokenExpire" src/

# ripgrep (더 빠름)
rg "tokenExpire" src/

# 특정 파일 타입만
grep -rn "tokenExpire" --include="*.jsp" src/
grep -rn "tokenExpire" --include="*.java" src/
```

| 옵션 | 의미 |
|------|------|
| `-r` | 하위 디렉토리 재귀 검색 |
| `-n` | 라인 번호 표시 |
| `-i` | 대소문자 무시 |
| `--include` | 파일 패턴 필터 |

### git log 검색

"이 값이 언제, 누가 넣었는지" 추적:

```bash
# 특정 문자열이 추가/삭제된 커밋 찾기
git log -S "tokenExpire = 300" -- "*.jsp"

# 특정 파일의 전체 변경 이력
git log --oneline -- "src/main/webapp/WEB-INF/jsp/crs/cms/mediopia_cms_pop.jsp"

# 특정 커밋에서 뭐가 바뀌었는지
git show <커밋해시>

# 특정 라인이 누가 마지막으로 수정했는지
git blame src/main/webapp/WEB-INF/jsp/crs/cms/mediopia_cms_pop.jsp
```

## 원칙 3: 범위 좁히기

검색 결과가 너무 많을 때:

!!! note "좁히기 전략"
    `"tokenExpire"` → 결과 100개?

    1. **파일 타입 필터**: `*.jsp`, `*.java`만
    2. **디렉토리 필터**: `src/main/` (target, node_modules 제외)
    3. **키워드 조합**: `"tokenExpire = "` (할당문만)
    4. **컨텍스트 확인**: 앞뒤 5줄 같이 보기 (`-C 5`)

## 실전 연습: 오늘 케이스를 직접 찾아보는 과정

업체 답변을 받은 시점으로 돌아가자. 이렇게 했어야 한다:

### Step 1: 키워드 추출 (30초)

!!! example "키워드 추출"
    업체 답변에서 코드 관련 키워드 뽑기:
    → `tokenExpire`, `JwtProvider`, `CMS_JWT_API_KEY`

### Step 2: 첫 번째 검색 (1분)

!!! example "첫 번째 검색"
    `Ctrl+Shift+F` → `"tokenExpire"`
    → `mediopia_cms_pop.jsp:10  int tokenExpire = 300;`
    → 찾았다. 300초.

끝이야? **아니다.** 여기서 한 단계 더.

### Step 3: 설정 파일도 확인 (1분)

!!! example "설정 파일도 확인"
    "300이 하드코딩인데, 혹시 프로퍼티에서 읽어오는 곳은 없나?"

    `Ctrl+Shift+F` → `"CMS_JWT"`
    → `Constants.java:450  CMS_JWT_EXPIRE = framework.getInt("framework.lxp.cms.jwt.expire")`
    → `framework.properties:316  framework.lxp.cms.jwt.expire=300`

    "프로퍼티에도 300이네. 그런데 JSP에서 Constants를 안 쓰고 하드코딩이네."

### Step 4: 변경 이력 확인 (2분)

!!! example "변경 이력 확인"
    `git log -S "tokenExpire = 300" -- "*.jsp"`
    → 최초 커밋: 2022-07-28, 메디오피아 개발자
    → 이후 변경 없음

    "처음부터 300으로 들어왔고, 아무도 안 바꿨다."

### Step 5: 업체에 회신 (1분)

!!! example "업체에 회신"
    "tokenExpire = 300초(5분)입니다.
    최초 세팅(2022-07-28)부터 이 값이었습니다.
    3600초로 변경 필요합니다."

**총 소요 시간: 5분 이내**

## 검색 시나리오별 전략

| 상황 | 검색 전략 |
|------|-----------|
| 변수명/클래스명을 알 때 | IDE 전체 검색 (Ctrl+Shift+F) |
| 에러 메시지가 있을 때 | 에러 메시지 문자열로 검색 |
| 설정값을 찾을 때 | `*.properties`, `*.yml`, `*.xml` 필터 |
| "누가 바꿨어?"가 궁금할 때 | `git log -S` 또는 `git blame` |
| 특정 API가 어디서 호출되는지 | 메서드명으로 검색 |
| DB 쿼리를 찾을 때 | `*Mapper*.xml` 필터 + SQL 키워드 |

## 검색할 때 흔한 실수

### 실수 1: 자연어로 검색

!!! danger "잘못된 예"
    "토큰 만료 시간 설정" → 결과 0개 (코드는 한글이 아니야)

!!! success "올바른 예"
    "tokenExpire" → 정확히 찾음

### 실수 2: 범위를 안 좁힘

!!! danger "잘못된 예"
    프로젝트 전체에서 "token" 검색 → 결과 5000개

!!! success "올바른 예"
    `src/main/` 폴더에서 "tokenExpire" 검색 → 결과 2개

### 실수 3: 한 곳 찾고 끝냄

!!! danger "잘못된 예"
    JSP에서 300 찾았다 → 끝!

!!! success "올바른 예"
    "Constants에도 있나? properties에도 있나? 다른 파일에도 있나?"
    → 전체 그림을 봐야 한다

### 실수 4: 검색 도구를 하나만 씀

!!! danger "잘못된 예"
    IDE 검색만 → "누가 바꿨는지"는 모름

!!! success "올바른 예"
    IDE 검색 + git log → 값도 찾고 이력도 확인
