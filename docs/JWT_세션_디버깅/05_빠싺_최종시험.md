## Part 5: 종합 시나리오

**Q17.** 새로운 외부 솔루션 연동 시 "로그인은 되는데 10분 후에 튕겨요"라는
버그 리포트가 들어왔다. 네가 디버깅하는 순서를 **5단계**로 말해봐.

**Q18.** 후배가 와서 이렇게 물어봤다:

> "CMS에서 401 에러가 나는데 뭐가 문제인지 모르겠어요"

네가 후배한테 할 **질문 5개**를 순서대로 말해봐.
(직접 해결하는 게 아니라, 후배가 스스로 찾게 유도하는 질문)

**Q19.** 오늘 문제 (tokenExpire = 300초)의 재발을 방지하려면
어떤 구조적 개선이 가능한지 **3가지** 제시해봐.
(단순히 "값을 바꾼다"는 답이 아니야)

**Q20.** 아래 코드의 문제점을 **3가지 이상** 찾아봐:

```java
<%
    int tokenExpire = 300;
    HashMap<String, Object> payload = new HashMap<String, Object>();
    payload.put("userId", UserBroker.getUserId(request));
    payload.put("userName", UserBroker.getUserName(request));
    payload.put("domain", "localhost");

    JwtProvider jwtProvider =
        new JwtProvider(SignatureAlgorithm.HS256,
            Constants.CMS_JWT_API_KEY, Constants.CMS_JWT_API_SECRET);

    String jwt = jwtProvider.generatePayloadToken(payload, tokenExpire);
%>
<html>
<body onload="javascript:document.authForm.submit();">
    <form name="authForm" action="<%=rUrl%>" method="post">
        <input type="hidden" name="jwtToken" value="Bearer <%=jwt%>" />
    </form>
</body>
</html>
```

힌트: 하드코딩, 보안, 설계, 유지보수 관점에서 각각 하나씩.
