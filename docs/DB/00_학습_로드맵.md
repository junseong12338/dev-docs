## 핵심 원칙

!!! abstract "핵심 원칙"
    1. 용어를 '안다'는 건 남에게 설명할 수 있다는 뜻이다
    2. 명령어를 '쓸 줄 안다'는 건 결과를 예측할 수 있다는 뜻이다
    3. DB를 '관리한다'는 건 터졌을 때 원인 찾고 고칠 수 있다는 뜻이다
    4. "돌아가요"는 0점. "왜 돌아가는지"를 설명해야 점수다.

## 우리가 분석할 실제 데이터

```
서버: knuLMS-DB01
DB: MariaDB (InnoDB)
스키마: lxpknu10

문제 테이블: tb_lms_exam_stare_paper_backup
  - 행 수: 2억 7,649만 건
  - 용량: 43GB (전체 DB 50GB의 86%)
  - 인덱스: 없음
  - COUNT(*) 소요: 27분
  - mysqldump 소요: 20분 (1,249초)

디스크: /data 196GB 중 134GB 사용 (72%)
백업: 매일 00:00 mysqldump → 14개월째 0바이트 (실패)
에러: "Disk is full" 229건 누적 (2024-06 ~ 2026-02)

원인:
  - jQuery 이벤트 핸들러 중복 등록 → AJAX N배 증폭
  - 루프 내 전체 백업 호출 → 학생수² × 문항수 폭발
  - 삭제 정책 없음 → 무한 축적

조치:
  - 코드 버그 수정 (BUG-1, BUG-2)
  - RENAME TABLE → 빈 테이블 생성
  - TRUNCATE → 43GB 확보
  - metadata lock 대기 → SHOW PROCESSLIST → KILL로 해결
```

이 숫자들이 뭘 의미하는지, 왜 이렇게 됐는지, 어떻게 대응하는지.
전부 씹어먹는다.

## 레벨 기준

!!! tip "레벨 기준"
    **Lv1**: "SELECT 쓸 줄 알아요"
    → 불합격. 그건 당연한 거야.

    **Lv2**: "DDL이랑 DML 차이 설명할 수 있어요"
    → 겨우 시작. 아직 멀었어.

    **Lv3**: "트랜잭션, 락, 인덱스 개념 알아요"
    → 개념은 아는구나. 근데 실전에서 써봤어?

    **Lv4**: "SHOW PROCESSLIST 보고 원인 찾을 수 있어요"
    → 이제 좀 쓸만하네. 근데 빈틈 있어.

    **Lv5**: "장애 터지면 혼자 대응 가능해요"
    → 드디어 괴물. 근데 여기서 끝이 아니야.

    **Lv무한**: "DB 서버 안 건드리고도 코드만 보고 DB 문제 예측 가능"
    → 이게 진짜 괴물이야.
