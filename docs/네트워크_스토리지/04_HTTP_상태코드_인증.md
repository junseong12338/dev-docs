# 04. HTTP 상태 코드 & 인증 - 에러의 언어를 읽어라

> 👹 "401이 뭔지 모르고 '안돼요'만 하면 Lv1이야.
> 에러 코드는 서버가 보내는 메시지야. 읽을 줄 알아야 해."

---

## HTTP 상태 코드 체계

```
1xx: 정보 (처리 중)
2xx: 성공 ✅
3xx: 리다이렉트 (다른 곳으로 가라)
4xx: 클라이언트 에러 (네가 잘못함) ❌
5xx: 서버 에러 (서버가 잘못함) 💥
```

---

## 오늘 만난 코드들

### 200 OK ✅
```
GET /knu9 → 200
"야, 너 요청한 거 여기 있어. 가져가."
```
- 토큰으로 bucket 목록 조회했을 때

### 201 Created ✅
```
PUT /knu9/file.mp4 → 201
"새로운 오브젝트 만들어줬어."
```
- S3에 파일 업로드 성공했을 때

### 400 Bad Request ❌
```
curl /auth/v1.0 → 400
"네 요청이 문법적으로 틀려. 뭔 말인지 모르겠어."
```
- Swift 토큰 재발급 시도했을 때 (인증 URL 형식 안 맞음)

### 401 Unauthorized ❌
```
PUT /knu9/file.mp4 → 401
"너 누군지 모르겠어. 인증 안 됨."
```
- 토큰에 쓰기 권한 없었을 때

### 404 Not Found ❌
```
GET /play/UUID/... → 404
"그런 거 없어."
```
- CMS 미등록 CDN URL 접근했을 때 (이 프로젝트의 시작!)

---

## 401 vs 403 차이 (중요!)

```
401 Unauthorized: "너 누구야? 인증부터 해."
  → 인증 자체가 안 됨
  → 토큰 만료, 키 틀림, 권한 없음

403 Forbidden: "너 누군지는 아는데, 권한 없어."
  → 인증은 됐는데 접근 거부
  → 읽기만 가능한데 쓰기 시도

오늘의 경우:
  GET → 200 (읽기 OK)
  PUT → 401 (쓰기 거부)
  → 사실 403이 더 정확하지만, 서버 구현마다 다름
  → 중요한 건: "인증은 됐는데 해당 작업은 안 됨"을 파악하는 것
```

## 인증(Authentication) vs 인가(Authorization)

> 👹 "이 두 개 구분 못 하면 보안 얘기 자격 없어."

!!! tip "인증 vs 인가"

    **인증 (Authentication) = "너 누구야?"**

    - ID/PW, API Key, 토큰
    - 신분증 확인

    **인가 (Authorization) = "너 이거 해도 돼?"**

    - 권한 체크 (읽기/쓰기/삭제)
    - 출입 허가증 확인

    **예시:**

    1. 호텔 체크인 (인증) -> 카드키 받음
    2. 3층 수영장 입장 (인가) -> VIP만 가능

    **오늘의 상황:**

    1. API Token으로 인증 -> 성공 (200)
    2. PUT 요청 -> 인가 실패 (읽기 전용 토큰)

## 오늘 사용한 인증 방식 총정리

| 서비스 | 인증 방식 | 정보 | 결과 |
|--------|----------|------|------|
| VOD 서버 SSH | ID + Password | tideflo / 1q2w3e4r!! | ✅ |
| VOD 서버 FTP | ID + Password | tideflo / 1q2w3e4r!! | ✅ |
| S3 (boto3) | Access Key + Secret Key | a1a59c... / edd018... | ✅ 읽기+쓰기 |
| S3 (WinSCP) | Access Key + Secret Key | 위와 동일 | ✅ 읽기+쓰기 |
| Swift (curl) | API Token | gAAAAABh6RIA... | ⚠️ 읽기만 |
| Swift (Cyberduck Keystone) | Keystone 인증 | S3 Key를 넣음 (불일치) | ❌ |

> 👹 **결론: 같은 스토리지인데 인증 방식에 따라 권한이 다를 수 있다.**
> S3 Key로는 읽기+쓰기 되는데, Swift Token으로는 읽기만 됐잖아.
> 이게 인증/인가의 실전이야.
