13. 마스터 플레이리스트 - Gamma¶

1. 마스터 플레이리스트가 뭐야?¶

HLS에서 여러 해상도(비트레이트)의 스트림을 하나로 묶는 상위 플레이리스트다.

플레이어는 이 파일 하나만 받으면, 네트워크 상태에 따라 알아서 적절한 해상도를 골라서 재생한다. 이게 ABR(Adaptive Bitrate Streaming)의 핵심이다.

graph TD
    A[master.m3u8<br/>마스터 플레이리스트] --> B[360p/playlist.m3u8]
    A --> C[480p/playlist.m3u8]
    A --> D[720p/playlist.m3u8]
    A --> E[1080p/playlist.m3u8]
    B --> B1[360p/seg_000.ts]
    B --> B2[360p/seg_001.ts]
    C --> C1[480p/seg_000.ts]
    C --> C2[480p/seg_001.ts]
    D --> D1[720p/seg_000.ts]
    D --> D2[720p/seg_001.ts]
    E --> E1[1080p/seg_000.ts]
    E --> E2[1080p/seg_001.ts]

왜 필요한가

미디어 플레이리스트만 있으면 해상도가 하나 고정이다. WiFi에서 5G로, 5G에서 3G로 네트워크가 바뀌어도 화질 전환이 안 돼. 마스터 플레이리스트가 있어야 플레이어가 상황에 맞게 스트림을 전환할 수 있다.

2. 마스터 플레이리스트 구조¶

#EXTM3U
#EXT-X-VERSION:3

#EXT-X-STREAM-INF:BANDWIDTH=800000,RESOLUTION=640x360,CODECS="avc1.42c01e,mp4a.40.2"
360p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=1400000,RESOLUTION=854x480,CODECS="avc1.42c01e,mp4a.40.2"
480p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=2800000,RESOLUTION=1280x720,CODECS="avc1.4d401f,mp4a.40.2"
720p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=5000000,RESOLUTION=1920x1080,CODECS="avc1.640028,mp4a.40.2"
1080p/playlist.m3u8

3. EXT-X-STREAM-INF 태그 해부¶

이 태그가 마스터 플레이리스트의 핵심이다. 각 속성을 뜯어보자:

속성 의미 예시
BANDWIDTH 이 스트림을 재생하는 데 필요한 최대 비트레이트(bps) 2800000 = 2.8Mbps
RESOLUTION 해상도 (가로x세로) 1280x720
CODECS 비디오/오디오 코덱 식별자 "avc1.4d401f,mp4a.40.2"
FRAME-RATE 프레임레이트 (선택) 30.000
AVERAGE-BANDWIDTH 평균 비트레이트 (선택) 2500000

BANDWIDTH¶

#EXT-X-STREAM-INF:BANDWIDTH=2800000,...
  • 단위: bps (bits per second). kbps나 Mbps 아니다.
  • 비디오 + 오디오 + 컨테이너 오버헤드를 전부 포함한 값이어야 한다.
  • 플레이어는 이 값과 현재 네트워크 대역폭을 비교해서 스트림을 선택한다.

BANDWIDTH 계산법

BANDWIDTH = 비디오 비트레이트 + 오디오 비트레이트 + 오버헤드(~10%)
예: 비디오 2500kbps + 오디오 128kbps = 2628kbps → 오버헤드 포함 약 2800kbps = 2800000bps

이 값을 실제 비트레이트보다 낮게 잡으면 버퍼링, 높게 잡으면 불필요하게 낮은 해상도 선택.

CODECS¶

CODECS="avc1.4d401f,mp4a.40.2"
코덱 문자열 의미
avc1.42c01e H.264 Baseline Profile, Level 3.0
avc1.4d401f H.264 Main Profile, Level 3.1
avc1.640028 H.264 High Profile, Level 4.0
mp4a.40.2 AAC-LC

CODECS를 왜 쓰는가

브라우저/플레이어가 지원하지 않는 코덱이면 아예 시도를 안 한다. CODECS 없으면 플레이어가 세그먼트를 받아보고 나서야 "이거 못 재생해" 하는데, CODECS 있으면 미리 판단해서 지원 가능한 스트림만 선택한다.

4. 멀티 해상도 HLS 만드는 FFmpeg 명령어¶

방법 1: 해상도별 개별 인코딩 후 마스터 수동 생성¶

# 360p 인코딩
ffmpeg -i input.mp4 \
  -c:v libx264 -preset medium \
  -b:v 800k -maxrate 856k -bufsize 1200k \
  -vf "scale=640:360" \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 96k -ac 2 \
  -hls_time 4 -hls_playlist_type vod \
  -hls_segment_filename "360p/segment_%03d.ts" \
  360p/playlist.m3u8

# 720p 인코딩
ffmpeg -i input.mp4 \
  -c:v libx264 -preset medium \
  -b:v 2500k -maxrate 2675k -bufsize 5350k \
  -vf "scale=1280:720" \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 128k -ac 2 \
  -hls_time 4 -hls_playlist_type vod \
  -hls_segment_filename "720p/segment_%03d.ts" \
  720p/playlist.m3u8

# 1080p 인코딩
ffmpeg -i input.mp4 \
  -c:v libx264 -preset medium \
  -b:v 5000k -maxrate 5350k -bufsize 10700k \
  -vf "scale=1920:1080" \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 192k -ac 2 \
  -hls_time 4 -hls_playlist_type vod \
  -hls_segment_filename "1080p/segment_%03d.ts" \
  1080p/playlist.m3u8

마스터 플레이리스트는 수동으로 작성:

#EXTM3U
#EXT-X-VERSION:3

#EXT-X-STREAM-INF:BANDWIDTH=928000,RESOLUTION=640x360
360p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=2800000,RESOLUTION=1280x720
720p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=5500000,RESOLUTION=1920x1080
1080p/playlist.m3u8

방법 2: FFmpeg 한 번에 멀티 출력 (추천)¶

ffmpeg -i input.mp4 \
  -filter_complex \
    "[0:v]split=3[v360][v720][v1080]; \
     [v360]scale=640:360[v360out]; \
     [v720]scale=1280:720[v720out]; \
     [v1080]scale=1920:1080[v1080out]" \
  -map "[v360out]" -map 0:a -c:v libx264 -preset medium \
    -b:v 800k -maxrate 856k -bufsize 1200k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 96k -ac 2 \
    -hls_time 4 -hls_playlist_type vod \
    -hls_segment_filename "360p/seg_%03d.ts" \
    360p/playlist.m3u8 \
  -map "[v720out]" -map 0:a -c:v libx264 -preset medium \
    -b:v 2500k -maxrate 2675k -bufsize 5350k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 128k -ac 2 \
    -hls_time 4 -hls_playlist_type vod \
    -hls_segment_filename "720p/seg_%03d.ts" \
    720p/playlist.m3u8 \
  -map "[v1080out]" -map 0:a -c:v libx264 -preset medium \
    -b:v 5000k -maxrate 5350k -bufsize 10700k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 192k -ac 2 \
    -hls_time 4 -hls_playlist_type vod \
    -hls_segment_filename "1080p/seg_%03d.ts" \
    1080p/playlist.m3u8

방법 1 vs 방법 2

항목 방법 1 (개별 인코딩) 방법 2 (한 번에)
디코딩 횟수 3번 (해상도마다) 1번
제어 용이성 각각 독립, 디버깅 쉬움 복잡하지만 효율적
마스터 플레이리스트 수동 작성 수동 작성
실무 선호도 스크립트로 자동화할 때 서버 리소스 절약할 때

어느 방식이든 마스터 플레이리스트는 직접 만들어야 한다. FFmpeg의 HLS muxer는 미디어 플레이리스트만 자동 생성해주고, 마스터 플레이리스트는 만들어주지 않는다.

5. 플레이어가 해상도를 선택하는 원리¶

sequenceDiagram
    participant P as 플레이어
    participant S as 서버

    P->>S: GET master.m3u8
    S-->>P: 마스터 플레이리스트 반환

    Note over P: 현재 대역폭 측정:<br/>3.5Mbps

    Note over P: BANDWIDTH 비교:<br/>360p=928k (OK)<br/>720p=2800k (OK)<br/>1080p=5500k (초과)

    Note over P: 720p 선택<br/>(재생 가능한 최고 해상도)

    P->>S: GET 720p/playlist.m3u8
    S-->>P: 미디어 플레이리스트

    P->>S: GET 720p/seg_000.ts
    Note over P: 재생 시작

    Note over P: 대역폭 계속 모니터링

    Note over P: 대역폭 6Mbps로 증가
    P->>S: GET 1080p/playlist.m3u8
    Note over P: 1080p로 전환

선택 알고리즘 (간략화)¶

  1. 초기 선택: 마스터 플레이리스트의 BANDWIDTH 값들 중, 현재 측정 대역폭의 70~80% 이하인 것 중 가장 높은 해상도 선택
  2. 업 전환: 대역폭이 충분히 높아지면 상위 해상도로 전환
  3. 다운 전환: 버퍼가 줄어들거나 대역폭이 떨어지면 하위 해상도로 전환
  4. 보수적 전략: 대부분의 플레이어는 업 전환보다 다운 전환을 빠르게 한다 (버퍼링 방지)

BANDWIDTH 값을 정확하게 잡아야 하는 이유

  • 너무 높게 잡으면: 충분한 대역폭인데도 낮은 해상도를 선택 → 화질 손해
  • 너무 낮게 잡으면: 대역폭 부족한데 높은 해상도 선택 → 버퍼링

6. 실전 해상도/비트레이트 조합¶

Apple과 주요 스트리밍 서비스에서 권장하는 조합이다:

해상도 비디오 비트레이트 오디오 비트레이트 총 BANDWIDTH 용도
640x360 800k 96k ~928k 3G, 저속 WiFi
854x480 1400k 128k ~1600k 4G
1280x720 2500k 128k ~2800k WiFi
1920x1080 5000k 192k ~5500k 고속 WiFi

최소 해상도는 반드시 포함

360p나 480p 같은 저해상도를 빼면, 저속 네트워크 환경에서 아예 재생이 안 된다. "우리 서비스는 고화질만 제공해"라는 사고방식은 사용자를 내쫓는 거다.

7. 주의사항 / 함정¶

함정 1: BANDWIDTH에 비디오 비트레이트만 넣기¶

# 틀린 예: 비디오만 2500kbps
#EXT-X-STREAM-INF:BANDWIDTH=2500000,RESOLUTION=1280x720

# 맞는 예: 비디오 + 오디오 + 오버헤드
#EXT-X-STREAM-INF:BANDWIDTH=2800000,RESOLUTION=1280x720

BANDWIDTH는 비디오 + 오디오 + 컨테이너 오버헤드를 전부 합한 값이다. 비디오만 넣으면 플레이어가 대역폭을 과소평가해서 버퍼링이 발생한다.

함정 2: 해상도 간 키프레임 정렬 안 맞음¶

모든 해상도에서 같은 GOP 크기, 같은 세그먼트 길이를 써야 한다. 안 그러면 해상도 전환 시 영상이 깨지거나 점프하는 현상이 생긴다.

# 모든 해상도에서 동일하게
-g 120 -keyint_min 120 -sc_threshold 0 -hls_time 4

함정 3: 마스터 플레이리스트 경로 오류¶

# 마스터 플레이리스트와 미디어 플레이리스트의 상대 위치가 맞아야 한다

# 마스터: /video/master.m3u8
# 미디어: /video/720p/playlist.m3u8

# 마스터 안에서의 경로:
720p/playlist.m3u8   ← 상대 경로 (올바름)

서버 이전이나 CDN 설정 변경 시 경로가 깨지는 게 흔하다. 배포 후 반드시 실제 재생 테스트를 해야 한다.

8. 정리¶

항목 내용
마스터 플레이리스트 여러 해상도의 미디어 플레이리스트를 묶는 상위 .m3u8
EXT-X-STREAM-INF 각 variant 스트림의 대역폭, 해상도, 코덱 정보
BANDWIDTH 비디오 + 오디오 + 오버헤드 합산 (bps 단위)
플레이어 선택 원리 현재 대역폭 < BANDWIDTH인 스트림 중 최고 해상도
키프레임 정렬 모든 해상도에서 동일한 GOP, 동일한 세그먼트 길이 필수
마스터 생성 FFmpeg이 자동 생성 안 함. 수동 작성 또는 스크립트

한 줄 정리

마스터 플레이리스트는 ABR의 핵심이고, BANDWIDTH를 정확히 계산하고, 모든 해상도에서 키프레임을 정렬하고, 마스터 파일은 직접 만들어야 한다.

확인 문제¶

다음 문제를 풀어봐. 답 못 하면 위에서 다시 읽어.

Q1. 비디오 비트레이트 2500kbps, 오디오 128kbps인 720p 스트림의 BANDWIDTH 값은 대략 얼마로 잡아야 하는지 계산해봐.

Q2. 플레이어의 현재 측정 대역폭이 2Mbps일 때, 아래 마스터 플레이리스트에서 어떤 해상도를 선택하는지 이유와 함께 답해봐. 

#EXT-X-STREAM-INF:BANDWIDTH=928000,RESOLUTION=640x360
#EXT-X-STREAM-INF:BANDWIDTH=2800000,RESOLUTION=1280x720
#EXT-X-STREAM-INF:BANDWIDTH=5500000,RESOLUTION=1920x1080

Q3. 360p, 720p, 1080p 3개 해상도의 HLS를 만들 때, 720p만 -g 120이고 나머지는 -g 60이면 어떤 문제가 생기는지 설명해봐.

Q4. FFmpeg의 HLS muxer가 자동 생성해주는 것과 직접 만들어야 하는 것을 구분해봐.

정답 보기

A1. 2500k + 128k = 2628kbps. 여기에 컨테이너 오버헤드 약 5~10%를 더하면 약 2760k~2890kbps. bps로 변환하면 약 2800000. BANDWIDTH=2800000이 적절하다.

A2. 360p(928kbps)를 선택한다. 플레이어는 보통 현재 대역폭의 70~80% 이하인 스트림을 선택한다. 2Mbps의 80%는 1.6Mbps인데, 720p(2.8Mbps)는 이를 초과한다. 360p(928kbps)는 1.6Mbps 이하이므로 360p가 선택된다.

A3. 해상도 간 키프레임 위치가 다르기 때문에 해상도 전환 시 문제가 생긴다. 720p는 4초마다 키프레임이고, 360p/1080p는 2초마다 키프레임이면 세그먼트 경계가 달라진다. 전환 시 영상이 점프하거나 깨지는 현상이 발생한다. 모든 해상도에서 동일한 GOP/hls_time을 써야 한다.

A4. FFmpeg이 자동 생성하는 것: 미디어 플레이리스트(.m3u8)와 세그먼트 파일(.ts). 직접 만들어야 하는 것: 마스터 플레이리스트. FFmpeg의 HLS muxer는 단일 해상도의 미디어 플레이리스트만 생성하고, 여러 해상도를 묶는 마스터 플레이리스트는 생성하지 않는다.
