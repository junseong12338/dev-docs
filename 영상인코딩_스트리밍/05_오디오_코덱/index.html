05. AAC - 소리도 압축한다 - Beta¶

1. 오디오 코덱이 왜 필요한가 - "이게 뭐야?"¶

영상 파일에는 영상(비디오)만 있는 게 아니야. 소리(오디오)도 들어있어.

CD 음질의 원본 오디오 데이터는 1초에 약 1.4Mbps야. 1시간이면 약 635MB. 영상 데이터에 비하면 작지만, 그냥 두면 쓸데없이 크다.

그래서 비디오에 H.264가 있듯, 오디오에는 AAC가 있어.

오디오 코덱의 역할

원본 오디오 데이터에서 사람이 못 듣는 소리를 제거하고, 반복 패턴을 압축해서 파일 크기를 줄이는 게 오디오 코덱이 하는 일이다.

비유하자면, 비디오 코덱이 눈에 보이지 않는 변화를 생략하는 거라면, 오디오 코덱은 귀에 들리지 않는 소리를 생략하는 거야.

손실 압축 vs 무손실 압축¶

구분 손실 압축 무손실 압축
원리 사람이 인지 못하는 정보 제거 데이터 패턴만 압축, 정보 손실 없음
코덱 AAC, MP3, Opus FLAC, ALAC
압축률 원본의 1/10~1/20 원본의 1/2~1/3
용도 스트리밍, 일반 재생 음악 제작, 보관용

스트리밍에서는 손실 압축

인코딩 파이프라인에서 오디오는 거의 100% 손실 압축(AAC)을 쓴다. 무손실은 파일이 너무 커서 스트리밍에 부적합해.

2. AAC vs MP3 비교 - "왜 AAC가 표준이 됐나"¶

MP3는 1993년에 나왔고, AAC는 1997년에 나왔어. AAC가 MP3의 후계자야.

항목 MP3 (MPEG-1 Audio Layer 3) AAC (Advanced Audio Coding)
출시 1993년 1997년
같은 비트레이트 음질 보통 MP3보다 좋음
128kbps 기준 고음 뭉개짐 깔끔한 고음 유지
최대 채널 2채널 (스테레오) 48채널 (5.1, 7.1 서라운드 가능)
최대 샘플레이트 48kHz 96kHz
HLS 스트리밍 비표준 공식 표준
브라우저 지원 대부분 지원 모든 브라우저 지원

같은 128kbps인데 왜 AAC가 더 좋냐?

AAC는 MP3보다 더 발전된 심리음향 모델을 사용해. 사람의 청각 특성을 더 정밀하게 분석해서, 같은 데이터 양으로 더 많은 청각 정보를 보존할 수 있어.

graph LR
    A[원본 오디오<br/>1,411 kbps] --> B{어떤 코덱?}
    B -->|MP3 128kbps| C[MP3 파일<br/>고음 손실 있음]
    B -->|AAC 128kbps| D[AAC 파일<br/>고음 잘 보존]
    B -->|AAC 192kbps| E[AAC 파일<br/>거의 원본급]

MP3는 죽은 코덱이 아니야

호환성 측면에서 MP3가 아직도 쓰이는 곳이 있어. 하지만 새로 만드는 서비스에서 MP3를 선택할 이유는 없다. HLS 표준이 AAC인 이상, AAC가 정답이야.

3. 샘플레이트와 비트레이트 - "소리 품질을 결정하는 숫자들"¶

샘플레이트 (Sample Rate)¶

소리는 파동이야. 이 파동을 디지털로 바꾸려면 초당 몇 번 측정(샘플링)할지 정해야 해.

샘플레이트 = 초당 샘플 수

44,100Hz(44.1kHz)면 초당 44,100번 소리를 측정한다는 뜻이야. 측정 횟수가 많을수록 원본 소리에 가까워지고, 파일은 커져.

샘플레이트 용도 주파수 범위
22.05kHz AM 라디오급, 음성 통화 ~11kHz까지
44.1kHz CD 음질, 음악 ~20kHz까지 (사람 청각 한계)
48kHz 영상 표준, 방송 ~24kHz까지
96kHz 스튜디오 녹음 ~48kHz까지

영상 인코딩에서는 48kHz

영상에서 오디오 샘플레이트는 48kHz가 업계 표준이야. 44.1kHz는 CD 음악 표준이고, 영상은 방송 규격인 48kHz를 따른다. 이거 섞어 쓰면 리샘플링 때문에 품질 떨어져.

비트레이트 (Bitrate)¶

비트레이트 = 초당 데이터 양

인코딩된 오디오가 초당 얼마나 많은 데이터를 사용하는지를 나타내. 높을수록 음질이 좋고, 파일이 커져.

비트레이트 음질 수준 용도
64kbps 라디오급, 음성 위주 팟캐스트, 음성 강의
128kbps 일반 음악 감상급 일반 스트리밍
192kbps 고음질 음악 스트리밍
256kbps 거의 원본급 고급 음악 서비스
320kbps MP3 최대, 원본에 가까움 다운로드 음원

비트레이트 높다고 무조건 좋은 거 아니야

128kbps AAC면 대부분의 사람이 원본과 구분 못 해. 192kbps 이상은 이어폰/헤드폰 좋은 사람이나 차이 느낄 수 있는 수준이야. 인코딩 비용 대비 효과를 생각해야 해.

샘플레이트 vs 비트레이트 차이¶

이 둘 헷갈리는 사람 많아. 확실히 구분하자.

구분 샘플레이트 비트레이트
뭐냐 초당 측정 횟수 초당 데이터 양
단위 Hz (kHz) bps (kbps)
결정하는 것 표현 가능한 주파수 범위 인코딩된 결과물의 품질/크기
비유 카메라 해상도 (얼마나 세밀하게 찍나) JPEG 압축률 (얼마나 압축하나)
원본 속성 원본 녹음 시 결정 인코딩 시 결정

4. 채널 - "모노, 스테레오, 서라운드"¶

채널 = 독립적인 오디오 트랙 수

귀 하나로 듣는 게 모노, 양쪽 귀로 듣는 게 스테레오야.

채널 설명 용도
모노 (1ch) 하나의 스피커 음성 통화, 간단한 알림음
스테레오 (2ch) 왼쪽 + 오른쪽 음악, 일반 영상, 대부분의 콘텐츠
5.1ch 5개 스피커 + 1개 서브우퍼 영화관, 홈시어터
7.1ch 7개 스피커 + 1개 서브우퍼 프리미엄 영화관

스트리밍에서 채널 선택

웹 스트리밍은 스테레오(2ch)가 기본이야. 5.1은 넷플릭스 같은 프리미엄 서비스에서나 쓰고, 일반 LMS나 강의 영상은 스테레오면 충분해. 강의 영상이라면 모노도 괜찮아. 채널이 늘면 데이터 양도 비례해서 늘어나니까.

5. 영상 인코딩에서 오디오 설정 - "코드로 보자"¶

FFmpeg으로 영상을 인코딩할 때 오디오 설정하는 법을 보자.

기본 오디오 인코딩¶

# 기본 AAC 인코딩
ffmpeg -i input.mp4 \
  -c:a aac \          # 오디오 코덱: AAC
  -b:a 128k \         # 오디오 비트레이트: 128kbps
  -ar 48000 \         # 샘플레이트: 48kHz
  -ac 2 \             # 채널 수: 2 (스테레오)
  output.mp4

# -c:a  = codec:audio (오디오 코덱 지정)
# -b:a  = bitrate:audio (오디오 비트레이트 지정)
# -ar   = audio rate (샘플레이트 지정)
# -ac   = audio channels (채널 수 지정)

용도별 오디오 설정 가이드¶

용도 코덱 비트레이트 샘플레이트 채널
강의 영상 AAC 128k 48000 2 (스테레오)
음성 위주 (팟캐스트) AAC 64k 48000 1 (모노)
음악 영상 AAC 192k~256k 48000 2 (스테레오)
고품질 영화 AAC 256k~320k 48000 2 또는 5.1 
ffmpeg -i input.mp4 \
  -c:v libx264 -crf 23 \
  -c:a aac -b:a 128k -ar 48000 -ac 2 \
  output.mp4

ffmpeg -i input.mp4 \
  -c:v libx264 -crf 23 \
  -c:a aac -b:a 64k -ar 48000 -ac 1 \
  output.mp4

ffmpeg -i input.mp4 \
  -c:v libx264 -crf 18 \
  -c:a aac -b:a 256k -ar 48000 -ac 2 \
  output.mp4

오디오 복사 (재인코딩 하지 않기)¶

오디오를 건드릴 필요 없을 때

원본 오디오가 이미 AAC이고 설정이 괜찮으면, 재인코딩하지 말고 그대로 복사해. 재인코딩은 품질 손실을 만들어.

# 비디오만 재인코딩하고 오디오는 그대로 복사
ffmpeg -i input.mp4 \
  -c:v libx264 -crf 23 \
  -c:a copy \          # 오디오 재인코딩 안 함, 원본 그대로 복사
  output.mp4

이거 실수하면 삽질해

-c:a copy로 오디오를 복사했는데 원본 오디오 코덱이 HLS에서 지원 안 되는 형식(예: PCM, AC3)이면 재생이 안 돼. HLS 출력이면 반드시 AAC인지 확인하고, 아니면 -c:a aac으로 변환해야 해.

6. 정리¶

항목 핵심
오디오 코덱 사람이 못 듣는 소리를 제거해서 파일 크기를 줄이는 기술
AAC MP3의 후계자, HLS 표준 오디오 코덱
AAC vs MP3 같은 비트레이트에서 AAC가 음질 더 좋음
샘플레이트 초당 측정 횟수, 영상에서는 48kHz 표준
비트레이트 초당 데이터 양, 128kbps면 일반 스트리밍 충분
채널 모노(1ch), 스테레오(2ch), 서라운드(5.1ch)
실전 설정 -c:a aac -b:a 128k -ar 48000 -ac 2

한 줄 정리

오디오 인코딩은 AAC, 48kHz, 128kbps, 스테레오가 표준 설정이고, 원본이 이미 AAC면 -c:a copy로 재인코딩을 피해라.

확인 문제¶

다음 문제를 풀어봐. 답 못 하면 위에서 다시 읽어.

Q1. 영상 인코딩에서 오디오 샘플레이트 표준은 44.1kHz일까, 48kHz일까? 그리고 왜?

Q2. AAC 128kbps와 MP3 128kbps가 있어. 같은 비트레이트인데 음질 차이가 나는 이유는 뭐야?

Q3. HLS 스트리밍용으로 영상을 인코딩하는데, 원본 오디오가 AC3(돌비 디지털) 코덱이야. -c:a copy로 복사해도 괜찮아?

Q4. 강의 영상을 인코딩할 때 오디오 비트레이트를 320kbps로 설정하면 어떤 문제가 있어?

정답 보기 **A1.** 48kHz. 44.1kHz는 CD 음악 표준이고, 영상은 방송 규격인 48kHz를 따른다. 두 샘플레이트를 섞어 쓰면 리샘플링이 발생해서 불필요한 품질 저하가 생긴다. **A2.** AAC가 MP3보다 **더 발전된 심리음향 모델**을 사용하기 때문이다. 사람의 청각 특성을 더 정밀하게 분석해서, 같은 데이터 양으로 더 많은 청각 정보를 보존할 수 있다. **A3.** 안 된다. HLS는 오디오 코덱으로 **AAC만 표준 지원**한다. AC3을 그대로 복사하면 대부분의 HLS 플레이어에서 재생이 안 된다. `-c:a aac`으로 변환해야 한다. **A4.** 기능적으로는 문제 없지만 **낭비**야. 강의 영상은 주로 음성이라서 128kbps AAC면 충분하다. 320kbps로 올려봤자 사람이 차이를 거의 못 느끼면서, 파일 크기와 대역폭만 2.5배 늘어난다. 비용 대비 효과가 없다.
