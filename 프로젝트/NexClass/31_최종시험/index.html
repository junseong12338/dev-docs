31. 최종시험 - Black Hole¶

시험 규칙

01~30 전 범위. 30문제. 100점 만점. 여기서 80점 못 넘기면 처음부터 다시 읽어.

항목 규칙
합격 기준 80점 이상
시간 제한 60분
오픈북 금지
검색 금지
부분 점수 서술형만
총점 100점

배점 구성¶

유형 문제 수 배점 소계
객관식 (4지선다) 10문제 2점 20점
단답형 8문제 3점 24점
코드 작성/분석 7문제 5점 35점
서술형 5문제 각 3~5점 21점
합계 30문제 100점

Part 1: 객관식 (10문제 x 2점 = 20점)¶

각 문제에서 정답 1개를 골라라.

Q1. [프로젝트 개요 / 01장] (2점)¶

NexClass 프로젝트의 핵심 목표로 가장 정확한 것은?

  • (A) Zoom API를 래핑하는 범용 화상회의 플랫폼을 구축한다
  • (B) LMS의 VirtualCampus API를 호환하는 자체 화상강의 솔루션으로 드롭인 교체한다
  • (C) LMS 시스템 전체를 Spring Boot로 재개발한다
  • (D) Zoom SDK를 내장하여 브라우저에서 직접 화상회의를 구현한다

Q2. [기술스택 / 02장] (2점)¶

NexClass가 RestTemplate에서 PATCH 메서드를 사용하기 위해 추가한 의존성은?

  • (A) spring-boot-starter-webflux
  • (B) httpclient5 (Apache HttpClient5)
  • (C) okhttp3
  • (D) spring-boot-starter-web-services

Q3. [JPA 엔티티 / 04장] (2점)¶

Lesson 엔티티에서 @PrePersist가 하는 역할은?

  • (A) 엔티티 삭제 전에 연관 데이터를 정리한다
  • (B) 엔티티가 처음 저장될 때 createdAt, updatedAt, syncStatus 기본값을 설정한다
  • (C) 엔티티 조회 시 Lazy Loading을 트리거한다
  • (D) 엔티티 수정 시 변경 감지(Dirty Checking)를 활성화한다

Q4. [REST API / 06장] (2점)¶

NexClass의 API 응답 형식 중 LMS 호환용 응답의 데이터 필드명은?

  • (A) data
  • (B) dataList
  • (C) datas
  • (D) result

NexClass SecurityConfig에서 SessionCreationPolicy를 IF_REQUIRED로 설정한 이유는?

Q19. [JWT 인증 / 09장] (5점)¶

아래는 JwtAuthenticationFilter의 핵심 로직이다. (가)~(다) 를 채워라.

@Override
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain filterChain)
                                throws ServletException, IOException {

    String header = request.getHeader("__(가)__");

    if (header == null || !header.startsWith("__(나)__")) {
        filterChain.doFilter(request, response);
        return;
    }

    String token = header.substring(__(다)__);

    if (jwtProvider.validateToken(token)) {
        String userCd = jwtProvider.getSubject(token);
        UsernamePasswordAuthenticationToken authentication =
            new UsernamePasswordAuthenticationToken(
                userCd, null, Collections.emptyList());
        SecurityContextHolder.getContext().setAuthentication(authentication);
    }

    filterChain.doFilter(request, response);
}
