26. 대시보드 Thymeleaf SSR¶

핵심개념¶

SSR이 뭐야?¶

SSR(Server-Side Rendering)은 서버에서 HTML을 완성해서 브라우저로 보내는 방식이야.

방식 동작 예시
SSR 서버가 HTML 완성 → 브라우저에 전달 Thymeleaf, JSP, Freemarker
CSR 서버는 JSON만 → 브라우저가 JS로 화면 그림 React, Vue, Angular
혼합 SSR로 기본 구조 + AJAX로 동적 데이터 NexClass 대시보드 방식

NexClass 대시보드는 SSR + AJAX 혼합이야

페이지 뼈대는 Thymeleaf(SSR)로 그리고, 날짜별 강의 목록 같은 동적 데이터는 AJAX로 가져와. 이게 관리 도구에 가장 적합한 패턴이야.

왜 Thymeleaf인가?¶

관리 도구 = SSR이 정답

  • 사용자: 관리자 1~2명 (트래픽 걱정 없음)
  • 목적: 데이터 조회/관리 (화려한 인터랙션 불필요)
  • 개발 속도: SPA 프레임워크 세팅 없이 바로 개발 가능
  • SEO: 관리 도구에 SEO 필요 없음 (하지만 SSR은 기본 지원)
  • 배포: 백엔드와 같은 서버에서 돌아감 (별도 빌드/배포 불필요)

React/Vue로 관리 도구 만드는 건 오버엔지니어링이야.

코드 분석¶

세션 기반 인증¶

대시보드는 JWT가 아니라 HTTP 세션으로 인증해.

// 로그인 처리
@PostMapping("/login")
public String login(@RequestParam String username,
                    @RequestParam String password,
                    HttpSession session, Model model) {
    // 설정 파일의 관리자 계정과 비교
    if (adminUsername.equals(username)
            && adminPassword.equals(password)) {
        // 세션에 사용자 정보 저장
        session.setAttribute("dashboardUser", username);
        return "redirect:/dashboard";
    }
    // 실패 시 에러 메시지와 함께 로그인 페이지로
    model.addAttribute("error",
        "아이디 또는 비밀번호가 올바르지 않습니다");
    return "dashboard/login";
}

// 모든 페이지에서 세션 체크
private boolean isLoggedIn(HttpSession session) {
    return session.getAttribute("dashboardUser") != null;
}

// 각 핸들러 메서드에서 수동 체크
@GetMapping
public String main(HttpSession session, Model model) {
    if (!isLoggedIn(session))
        return "redirect:/dashboard/login";
    // ...
}

수동 세션 체크의 한계

매 핸들러마다 isLoggedIn(session) 체크를 반복하고 있어. 이건 DRY 위반이야. 하나라도 빠트리면 인증 없이 접근 가능.

개선 방법: HandlerInterceptor로 /dashboard/** 경로 전체에 자동 체크. 하지만 현재 규모(페이지 7개)에서는 이 정도면 충분해. YAGNI.

메인 대시보드 (main.html)¶

메인 대시보드는 요약 카드 6개 + 강의 테이블 + 최근 동기화로 구성돼.

@GetMapping
public String main(HttpSession session, Model model) {
    if (!isLoggedIn(session))
        return "redirect:/dashboard/login";

    // 요약 카드 데이터 (count 쿼리)
    model.addAttribute("lessonCount",
        lessonRepository.count());
    model.addAttribute("attendanceCount",
        attendanceRepository.count());
    model.addAttribute("recordingCount",
        recordingRepository.count());
    model.addAttribute("hostCount",
        hostPoolRepository.count());
    model.addAttribute("userCount",
        userRepository.count());
    model.addAttribute("courseCount",
        courseRepository.count());

    // 최근 강의 5건 (정렬 + 제한)
    List<Lesson> recentLessons = lessonRepository.findAll()
        .stream()
        .sorted((a, b) -> /* createdAt 역순 정렬 */)
        .limit(5)
        .collect(Collectors.toList());
    model.addAttribute("recentLessons", recentLessons);

    // 최근 동기화 3건
    List<SyncLog> recentSyncs = syncLogRepository.findAll()
        .stream()
        .sorted((a, b) -> /* startedAt 역순 정렬 */)
        .limit(3)
        .collect(Collectors.toList());
    model.addAttribute("recentSyncs", recentSyncs);

    return "dashboard/main";  // templates/dashboard/main.html 렌더링
}

NexClass 대시보드에서 사용하는 Thymeleaf 문법 정리.

문법 용도 예시
th:text 텍스트 출력 th:text="${lessonCount}"
th:each 리스트 반복 th:each="sync : ${recentSyncs}"
th:if 조건부 렌더링 th:if="${sync.syncStatus == 'COMPLETED'}"
th:unless 조건 반전 th:unless="${student.absent}"
th:href 링크 바인딩 th:href="@{/dashboard/lessons}"
th:attr 속성 동적 설정 th:attr="onclick=..."
th:replace fragment 삽입 th:replace="~{fragments/header :: nav('main')}"
th:classappend CSS 클래스 추가 th:classappend="${menu == 'main'} ? 'active'"
#temporals 날짜 포맷 #temporals.format(entry.joinTime, 'HH:mm:ss')
#lists 리스트 유틸 #lists.isEmpty(recentSyncs)

fragment 파라미터의 역할

nav('lessons')에서 'lessons'이 menu 변수로 들어가. 그러면 th:classappend="${menu == 'lessons'} ? 'active'"가 작동해서 현재 페이지의 사이드바 메뉴가 하이라이트돼.

강의 관리 - AJAX 페이지네이션 (lessons.html)¶

강의 목록은 SSR이 아니라 AJAX로 동적 로딩해. 왜?

날짜별 필터링이 필요하니까

날짜를 바꿀 때마다 페이지를 새로고침하면 느려. AJAX로 데이터만 갈아끼우는 게 훨씬 빠르고 UX가 좋아.

@GetMapping("/api/lessons/bydate")
@ResponseBody  // JSON 반환 (Thymeleaf 아님!)
public Map<String, Object> getLessonsForManagement(
        @RequestParam String date,
        @RequestParam(defaultValue = "0") int page,
        @RequestParam(defaultValue = "20") int size,
        HttpSession session) {

    // 세션 체크 (AJAX라도 인증 필수)
    if (!isLoggedIn(session))
        return Map.of("success", false,
                      "message", "로그인 필요");

    // 날짜 범위 필터링
    LocalDate targetDate = LocalDate.parse(date);
    LocalDateTime dayStart = targetDate.atStartOfDay();
    LocalDateTime dayEnd = targetDate.plusDays(1).atStartOfDay();

    // 메모리에서 필터링 + 정렬 + 페이징
    List<Lesson> allLessons = lessonRepository.findAll()
        .stream()
        .filter(l -> /* 해당 날짜 범위 */)
        .sorted(/* 시작시간 순 */)
        .collect(Collectors.toList());

    // 페이지 계산
    int total = allLessons.size();
    int totalPages = (int) Math.ceil((double) total / size);
    List<Lesson> pageList = allLessons
        .subList(fromIndex, toIndex);

    return Map.of(
        "success", true,
        "lessons", lessons,
        "total", total,
        "page", page,
        "totalPages", totalPages
    );
}

function loadLessons() {
    var dateStr = formatDate(currentDate);
    updateDateDisplay();

    // 로딩 스켈레톤 표시
    lessonBody.innerHTML = loadingState();

    // Fetch API로 JSON 요청
    fetch('/dashboard/api/lessons/bydate'
          + '?date=' + dateStr
          + '&page=' + currentPage
          + '&size=' + pageSize)
        .then(function(r) { return r.json(); })
        .then(function(data) {
            if (!data.success) {
                lessonBody.innerHTML =
                    emptyState('오류: ' + data.message);
                return;
            }
            if (data.lessons.length === 0) {
                lessonBody.innerHTML =
                    emptyState('해당 날짜에 강의가 없습니다');
                return;
            }
            // 테이블 행 동적 생성
            var html = '';
            data.lessons.forEach(function(l) {
                html += '<tr>';
                html += '<td>' + (l.lessonName || '-') + '</td>';
                // ...
                html += '</tr>';
            });
            lessonBody.innerHTML = html;

            // 페이지네이션 업데이트
            if (data.totalPages > 1) {
                paginationWrap.style.display = '';
                pageInfo.textContent =
                    '총 ' + data.total + '건';
            }
        });
}

출결 현황 - 순수 SSR (attendance.html)¶

출결 현황은 AJAX 없이 순수 SSR이야. 왜?

한 번 렌더링하면 데이터가 안 변하니까

출결 데이터는 한 강의에 대한 고정된 결과야. 날짜 필터처럼 실시간으로 바뀌는 게 아니라서 서버에서 한 번 렌더링하면 충분해.

<!-- 학생별 출결 요약 (서버에서 완성된 HTML) -->
<tr th:each="student : ${students}"
    th:unless="${student.absent}"
    class="summary-row">
    <td>
        <span class="nc-text-secondary"
              th:text="${student.userCd} ?: '-'"></span>
    </td>
    <td>
        <span style="font-weight:600"
              th:text="${student.userName}"></span>
    </td>
    <td>
        <!-- 초 → 분/초 변환 -->
        <span th:text="${student.totalDuration / 60} + '분'">
        </span>
        <span th:text="' ' + ${student.totalDuration % 60}
                        + '초'"></span>
    </td>
    <td>
        <span th:text="${student.entryCount}"></span>
    </td>
    <td>
        <!-- 외부 참가자 / 출석 뱃지 -->
        <span class="nc-badge nc-badge-muted"
              th:if="${student.external}">외부</span>
        <span class="nc-badge nc-badge-info"
              th:unless="${student.external}">출석</span>
    </td>
</tr>

세션 인증 vs JWT 인증¶

NexClass는 두 가지 인증이 공존해.

비교 항목 세션 인증 (대시보드) JWT 인증 (API)
대상 관리자 (브라우저) LMS (서버 간 통신)
저장 위치 서버 메모리 (HttpSession) 클라이언트 (Authorization 헤더)
상태 Stateful (서버가 세션 관리) Stateless (서버에 상태 없음)
로그아웃 session.invalidate() 토큰 만료 대기 (서버에서 무효화 불가)
적용 경로 /dashboard/** /v2/api/**
구현 HttpSession.setAttribute() JwtAuthenticationFilter

왜 대시보드에 JWT 안 쓰는 거야?

JWT는 브라우저에서 쓰기 불편해.

  • 매 요청마다 Authorization 헤더를 수동으로 붙여야 함
  • localStorage에 저장하면 XSS 공격에 취약
  • 로그아웃 즉시 무효화 불가능

브라우저 기반 인증은 쿠키+세션이 정석이야. JWT는 서버 간(M2M) 통신에 적합해.

주의사항¶

Thymeleaf 예약어 충돌

Thymeleaf에서 session은 예약어야. th:each의 반복 변수명으로 session을 쓰면 에러 나.

<!-- 금지 - session은 예약어 -->
<tr th:each="session : ${sessions}">

<!-- 올바름 - entry로 변수명 변경 -->
<tr th:each="entry : ${student.entryList}">

전체 조회 후 메모리 필터링

현재 코드는 findAll() 후 스트림으로 필터링해.

List<Lesson> allLessons = lessonRepository.findAll()
    .stream()
    .filter(l -> /* 날짜 범위 */)
    .collect(Collectors.toList());

데이터가 수백 건이면 괜찮지만, 수만 건이면 메모리 이슈 발생. 규모가 커지면 DB 쿼리 레벨에서 필터링하는 게 맞아. 현재 규모(강의 ~351건)에서는 문제없어. YAGNI.
