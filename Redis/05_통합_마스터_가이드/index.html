1. Quick Reference (빠른 참조)¶

1.1 Redis 핵심 개념 한 장 요약¶

Redis 핵심 개념 요약

Redis = 메모리 기반 초고속 Key-Value 저장소

특징:

  • In-Memory: RAM에 저장 → 초고속
  • Key-Value: 단순한 데이터 구조
  • TTL 지원: 자동 만료
  • Pub/Sub: 실시간 메시징

데이터 타입:

  • String: 단순 문자열 (세션 ID 등)
  • Hash: 필드-값 쌍 (사용자 정보 등)
  • List: 순서 있는 목록 (알림 목록 등)
  • Set: 중복 없는 집합 (팔로워 등)
  • Sorted Set: 점수 정렬 (랭킹 등)

주요 용도:

  • 세션 관리 (중복 로그인 방지)
  • 캐싱 (자주 조회하는 데이터)
  • 실시간 알림 (Pub/Sub)
  • Rate Limiting (API 호출 제한)

1.2 프로젝트별 Redis 사용 요약¶

프로젝트별 Redis 요약

KNU10WebService (HUB)

  • 용도: 중복 로그인 방지
  • 키: KNU10-HUB:{userId} → sessionId
  • TTL: 24시간 (슬라이딩)
  • DB: 0 (기본)
  • 설정: application.properties
  • 핵심 클래스: RedisUtil, SessionUtil

LXP-KNU10 (LMS)

  • 용도: 세션관리 + 실시간 알림
  • 키: KNU10-LMS:{module}:{key}
  • 모듈: exam, lesson, user
  • DB: 1
  • 설정: context-common.xml
  • 핵심 클래스: RedisComponent, *RedisPublisher
  • Pub/Sub: 시험종료, 학습중복, 로그인중복

LXP-SYNC

  • Redis 직접 사용 안 함
  • 배치성 데이터 동기화 담당

1.3 자주 사용하는 Redis 명령어¶

자주 사용하는 명령어

기본 명령어 (Redis CLI):

SELECT 1              # DB 1로 전환
KEYS KNU10-*          # 패턴 검색
GET key               # 값 조회
SET key value         # 값 저장
DEL key               # 삭제
TTL key               # 남은 TTL 확인
EXPIRE key 3600       # TTL 설정 (1시간)
FLUSHDB               # 현재 DB 전체 삭제 (주의!)

Java 코드:

// HUB
RedisUtil.addKeyValue("userId", "sessionId");
RedisUtil.getValueByKey("userId");
RedisUtil.removeByKeyAndValue("userId", "sessionId");

// LMS
redisComponent.setString("key", "value", "exam");
redisComponent.getString("key", "exam");
redisComponent.delete("key", "exam");

3. 코드 템플릿¶

3.1 HUB 스타일 (Static Utility)¶

/**
 * HUB 스타일 Redis 유틸리티 템플릿
 *
 * 특징:
 * - Static 메서드
 * - 어디서든 바로 호출 가능
 * - 단순한 세션 관리에 적합
 */
@Component
public class RedisUtil {

    private static RedisTemplate<String, String> redisTemplate;
    public static String REDIS_KEY_PREFIX;

    @Value("${spring.redis.key-prefix}")
    public void setPrefix(String prefix) {
        REDIS_KEY_PREFIX = prefix;
    }

    @Autowired
    public RedisUtil(RedisTemplate<String, String> template) {
        redisTemplate = template;
    }

    // === 저장 ===
    public static void set(String key, String value, long ttlHours) {
        redisTemplate.opsForValue().set(REDIS_KEY_PREFIX + key, value);
        redisTemplate.expire(REDIS_KEY_PREFIX + key, ttlHours, TimeUnit.HOURS);
    }

    // === 조회 (슬라이딩 세션) ===
    public static String get(String key) {
        String fullKey = REDIS_KEY_PREFIX + key;
        String value = redisTemplate.opsForValue().get(fullKey);
        if (value != null) {
            redisTemplate.expire(fullKey, 24, TimeUnit.HOURS);  // TTL 갱신
        }
        return value != null ? value : "empty";
    }

    // === 조건부 삭제 ===
    public static void deleteIfMatch(String key, String expectedValue) {
        String fullKey = REDIS_KEY_PREFIX + key;
        String currentValue = redisTemplate.opsForValue().get(fullKey);
        if (expectedValue.equals(currentValue)) {
            redisTemplate.delete(fullKey);
        }
    }
}

// 사용 예시
public void example() {
    RedisUtil.set("user123", "session_abc", 24);
    String session = RedisUtil.get("user123");
    RedisUtil.deleteIfMatch("user123", "session_abc");
}

3.2 LMS 스타일 (Component + Module)¶

/**
 * LMS 스타일 모듈별 Redis 컴포넌트 템플릿
 *
 * 특징:
 * - Bean 주입 방식
 * - 모듈별 키 네임스페이스 분리
 * - 다양한 데이터 타입 지원
 */
@Component
public class ModuleRedisComponent {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    @Resource(name = "redisTemplate")
    private ValueOperations<String, String> stringOps;

    @Resource(name = "redisTemplate")
    private HashOperations<String, String, Object> hashOps;

    private String site = "MY-APP";  // 사이트 식별자
    private String module = "myModule";  // 모듈명

    // === 키 생성 ===
    private String buildKey(String key) {
        return String.format("%s:%s:%s", site, module, key);
    }

    // === String 저장/조회 ===
    public void setString(String key, String value, long ttlSeconds) {
        stringOps.set(buildKey(key), value, ttlSeconds, TimeUnit.SECONDS);
    }

    public String getString(String key) {
        try {
            return stringOps.get(buildKey(key));
        } catch (Exception e) {
            return null;
        }
    }

    // === Hash 저장/조회 ===
    public void setHash(String key, Map<String, Object> data, long ttlSeconds) {
        String fullKey = buildKey(key);
        hashOps.putAll(fullKey, data);
        redisTemplate.expire(fullKey, ttlSeconds, TimeUnit.SECONDS);
    }

    public Object getHashField(String key, String field) {
        try {
            return hashOps.get(buildKey(key), field);
        } catch (Exception e) {
            return null;
        }
    }

    // === 공통 ===
    public boolean exists(String key) {
        return redisTemplate.hasKey(buildKey(key));
    }

    public void delete(String key) {
        redisTemplate.delete(buildKey(key));
    }
}

// 사용 예시
@Service
public class MyService {
    @Autowired
    private ModuleRedisComponent redis;

    public void example() {
        redis.setString("user:123", "data", 3600);
        String data = redis.getString("user:123");
        redis.delete("user:123");
    }
}

3.3 Pub/Sub 템플릿¶

// === Publisher ===
@Service
public class EventPublisher {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    public void publish(String channel, Object message) {
        redisTemplate.convertAndSend(channel, message);
    }
}

// === Subscriber ===
@Service
public class EventSubscriber implements MessageListener {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    @Override
    public void onMessage(Message message, byte[] pattern) {
        try {
            Object event = redisTemplate.getValueSerializer()
                                        .deserialize(message.getBody());

4. 트러블슈팅 가이드¶

4.1 자주 발생하는 문제¶

문제 1: 연결 실패

증상: Connection refused, Timeout

확인 사항:

  • Redis 서버 실행 여부 (ping 테스트)
  • 방화벽 설정 (6379 포트 열림 여부)
  • 호스트/포트 설정 확인
  • 비밀번호 일치 여부

해결: telnet 133.186.251.121 6379로 연결 테스트 → Redis CLI로 직접 연결 테스트 → application.properties 설정 확인

문제 2: 세션 불일치

증상: 로그인했는데 계속 로그아웃됨

확인 사항:

  • Redis 키 값 확인 (저장된 세션 ID)
  • 현재 세션 ID 확인
  • 다른 곳에서 로그인했는지 확인
  • TTL 만료 여부

디버깅: GET "KNU10-HUB:userId" → 로그에서 세션 ID 확인 → TTL "KNU10-HUB:userId"

문제 3: 메모리 부족

증상: OOM, 저장 실패

확인 사항:

  • INFO memory 명령으로 메모리 사용량 확인
  • TTL 없는 키 존재 여부
  • 대용량 데이터 저장 여부

해결: TTL 없는 키에 TTL 설정 → 불필요한 키 삭제 → 메모리 정책 설정 (maxmemory-policy)

문제 4: 역직렬화 실패

증상: ClassCastException, SerializationException

확인 사항:

  • 클래스 변경 여부 (필드 추가/삭제)
  • serialVersionUID 일치 여부
  • 직렬화 방식 일치 여부

해결: 기존 데이터 삭제 후 재저장 → JSON 직렬화로 변경 권장 → 클래스 변경 시 버전 관리

4.2 디버깅 명령어¶

# Redis CLI 접속
redis-cli -h 133.186.251.121 -p 6379 -a medi-redis8500

# DB 선택
SELECT 0  # HUB
SELECT 1  # LMS

# 키 검색
KEYS *               # 모든 키 (주의: 운영에서 사용 금지)
KEYS KNU10-HUB:*     # 패턴 검색
SCAN 0 MATCH KNU10-* COUNT 100  # 안전한 검색 (운영용)

# 값 확인
GET "KNU10-HUB:user123"
TYPE "KNU10-HUB:user123"  # 데이터 타입 확인
TTL "KNU10-HUB:user123"   # 남은 TTL (초)

# Hash 확인
HGETALL "KNU10-LMS:exam:key"

# 삭제

# 서버 정보

# 모니터링

5. 운영 가이드¶

5.1 모니터링 포인트¶

모니터링 포인트

1. 메모리 사용량 - used_memory / maxmemory / 경고: 80% 이상 시 주의

2. 연결 수 - connected_clients / 정상: 연결 풀 크기 x WAS 수 이내 / 비정상: 급격한 증가 = 연결 누수 의심

3. 명령 처리량 - instantaneous_ops_per_sec: 초당 처리 명령 / 갑작스러운 증가/감소 모니터링

4. 키 개수 - DBSIZE: 현재 DB의 키 개수 / 지속적 증가 = TTL 미설정 키 의심

5. 지연 시간 - 평균 응답 시간 1ms 이내가 정상 / 10ms 이상 = 병목 의심

5.2 운영 체크리스트¶

운영 체크리스트

일일 체크:

  • 메모리 사용량 확인
  • 연결 수 정상 범위 확인
  • 에러 로그 확인

주간 체크:

  • 키 개수 추이 확인
  • TTL 없는 키 존재 여부 확인
  • 슬로우 로그 확인 (SLOWLOG GET 10)

배포 전 체크:

  • 직렬화 클래스 변경 여부 확인
  • 키 네이밍 변경 영향 확인
  • TTL 설정 변경 영향 확인

장애 대응:

  • Redis 연결 실패 시 → 애플리케이션 동작 확인
  • 메모리 부족 시 → TTL 없는 키 정리
  • 성능 저하 시 → 슬로우 쿼리 확인
