1. MyBatisк°Җ лӯҗм•ј?В¶

SQL Mapper н”„л Ҳмһ„мӣҢнҒ¬лӢӨ. Java мҪ”л“ңм—җм„ң SQLмқ„ м§Ғм ‘ л¬ёмһҗм—ҙлЎң м§ңлҠ” лҢҖмӢ , XML нҢҢмқјм—җ SQLмқ„ л¶„лҰ¬н•ҙм„ң мһ‘м„ұн•ҳкі , Javaм—җм„ңлҠ” л©”м„ңл“ңл§Ң нҳём¶ңн•ҳл©ҙ лҗҳкІҢ л§Ңл“  кұ°м•ј.

// MyBatis м—Ҷмқҙ (мҲңмҲҳ JDBC)

PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, "U001");

2.3 #{} vs ${} - мқҙкұ° лӘЁлҘҙл©ҙ ліҙм•ҲмӮ¬кі  лӮңлӢӨВ¶

#{} vs ${}

#{value}

  • PreparedStatementмқҳ ? нҢҢлқјлҜён„°лЎң л°”мқёл”©
  • SQL Injection л°©м–ҙлҗЁ
  • к°’(value)м—җл§Ң мӮ¬мҡ©

${value}

  • л¬ёмһҗм—ҙ к·ёлҢҖлЎң м№ҳнҷҳ (л¬ёмһҗм—ҙ кІ°н•©)
  • SQL Injectionм—җ м·Ём•Ҫ!
  • н…Ңмқҙлё”лӘ…, м»¬лҹјлӘ… л“ұ кө¬мЎ°м—җ мӮ¬мҡ©

мӢӨм ңлЎң лӯҗк°Җ лӢӨлҘём§Җ:

-- #{userNo}лЎң л„ҳкё°л©ҙ (userNo = "U001")
SELECT * FROM TB_USER WHERE USER_NO = ?
-- PreparedStatementк°Җ ?м—җ 'U001'мқ„ м•Ҳм „н•ҳкІҢ л°”мқёл”©

-- ${userNo}лЎң л„ҳкё°л©ҙ (userNo = "U001")
SELECT * FROM TB_USER WHERE USER_NO = U001
-- л¬ёмһҗм—ҙмқҙ к·ёлҢҖлЎң SQLм—җ л°•нһҳ

SQL Injection кіөкІ© мӢңлӮҳлҰ¬мҳӨ:

-- м•…мқҳм Ғ мһ…л Ҙ: userNo = "U001' OR '1'='1"

-- #{userNo} мӮ¬мҡ© мӢң
SELECT * FROM TB_USER WHERE USER_NO = 'U001'' OR ''1''=''1'
-- вҶ’ мқҙмҠӨмјҖмқҙн”„ мІҳлҰ¬лҗЁ. кіөкІ© мӢӨнҢЁ.

-- ${userNo} мӮ¬мҡ© мӢң
SELECT * FROM TB_USER WHERE USER_NO = U001' OR '1'='1'
-- вҶ’ мЎ°кұҙмқҙ н•ӯмғҒ м°ё. м „мІҙ мӮ¬мҡ©мһҗ лҚ°мқҙн„° мң м¶ң!

мӣҗм№ҷ: к°’м—җлҠ” #{}, кө¬мЎ°(н…Ңмқҙлё”лӘ…/м»¬лҹјлӘ…/ORDER BY)м—җл§Ң ${} мӮ¬мҡ©. ${}лҘј м“ё л•ҢлҠ” л°ҳл“ңмӢң нҷ”мқҙнҠёлҰ¬мҠӨнҠё кІҖмҰқ.

3. лҸҷм Ғ SQLВ¶

SQLмқ„ мЎ°кұҙм—җ л”°лқј лӢӨлҘҙкІҢ мғқм„ұн•ҳлҠ” кё°лҠҘмқҙлӢӨ. Javaм—җм„ң ifл¬ёмңјлЎң SQL л¬ёмһҗм—ҙмқ„ мЎ°лҰҪн•ҳлҚҳ м§ҖмҳҘмқ„ MyBatisк°Җ н•ҙкІ°н•ҙмӨ¬лӢӨ.

3.1 if нғңк·ёВ¶

<select id="selectUserList" resultType="UserVO">

  • test мҶҚм„ұм—җ Java мЎ°кұҙмӢқмқ„ м“ҙлӢӨ.
  • мЎ°кұҙмқҙ м°ёмқј л•Ңл§Ң н•ҙлӢ№ SQL мЎ°к°Ғмқҙ нҸ¬н•ЁлҗңлӢӨ.
  • мЈјмқҳ: test м•Ҳм—җм„ң л¬ёмһҗм—ҙ л№„көҗлҠ” ==мқҙ м•„лӢҲлқј .equals() лҳҗлҠ” != мӮ¬мҡ©.

3.2 choose / when / otherwiseВ¶

Javaмқҳ if - else if - elseмҷҖ к°ҷлӢӨ.

  • м•„л¬ҙкІғлҸ„ м•Ҳ л§һмңјл©ҙ <otherwise> мӢӨн–ү.

3.4 foreach нғңк·ё (н•өмӢ¬!)В¶

лҰ¬мҠӨнҠёлӮҳ л°°м—ҙмқ„ мҲңнҡҢн•ҳл©ҙм„ң SQLмқ„ л°ҳліө мғқм„ұн•ҳлҠ” нғңк·ё. Batch мІҳлҰ¬мқҳ н•өмӢ¬мқҙлӢӨ.

<select id="selectUserByNos" resultType="UserVO">
    SELECT * FROM TB_USER
    WHERE USER_NO IN
    <foreach collection="list" item="userNo"
             open="(" separator="," close=")">
        #{userNo}
    </foreach>
</select>

мӢӨн–ү кІ°кіј: 

SELECT * FROM TB_USER WHERE USER_NO IN ('U001', 'U002', 'U003')

foreach мҶҚм„ұ мғҒм„ё:

мҶҚм„ұ м„ӨлӘ… мҳҲмӢң
collection мҲңнҡҢн•  лҢҖмғҒ. Listл©ҙ "list", Mapмқҳ keyл©ҙ keyлӘ… collection="list"
item к°Ғ мҡ”мҶҢлҘј лӢҙмқ„ ліҖмҲҳлӘ… item="userNo"
index нҳ„мһ¬ мқёлҚұмҠӨ (0л¶Җн„°) index="idx"
open л°ҳліө мӢңмһ‘ м „ м¶”к°Җн•  л¬ёмһҗм—ҙ open="("
close л°ҳліө лҒқлӮң нӣ„ м¶”к°Җн•  л¬ёмһҗм—ҙ close=")"

4.1 лӢЁкұҙ INSERT vs Batch INSERTВ¶

лӢЁкұҙ INSERT vs Batch INSERT (1000кұҙ кё°мӨҖ)

лӢЁкұҙ INSERT

// Java лЈЁн”„:
for (User user : userList) {
    mapper.insertUser(user);  // DB мҷ•ліө 1000лІҲ

мӢӨн–үлҗҳлҠ” SQL: INSERT л¬ёмқҙ 1000к°ң к°Ғк°Ғ мӢӨн–үлҗЁ

  • мҶҢмҡ” мӢңк°„: лҠҗлҰј (л„ӨнҠёмӣҢнҒ¬ мҷ•ліө * 1000)

Batch INSERT (foreach мӮ¬мҡ©)

// Java:
mapper.insertUserBatch(userList);  // DB мҷ•ліө 1лІҲ

мӢӨн–үлҗҳлҠ” SQL: VALUES (), (), () лЎң н•ң лІҲм—җ 1000кұҙ INSERT

  • DB мҷ•ліө: 1нҡҢ
  • мҶҢмҡ” мӢңк°„: л№ лҰ„ (л„ӨнҠёмӣҢнҒ¬ мҷ•ліө * 1)

4.3 INSERT ON DUPLICATE KEY UPDATEВ¶

MySQL м „мҡ© л¬ёлІ•мқҙлӢӨ. INSERT мӢңлҸ„ вҶ’ нӮӨ мӨ‘ліөмқҙл©ҙ UPDATEлЎң м „нҷҳ.

-- кё°ліё л¬ёлІ•
INSERT INTO н…Ңмқҙлё” (м»¬лҹј1, м»¬лҹј2, м»¬лҹј3)
VALUES ('к°’1', 'к°’2', 'к°’3')
ON DUPLICATE KEY UPDATE
    м»¬лҹј2 = VALUES(м»¬лҹј2),
    м»¬лҹј3 = VALUES(м»¬лҹј3)

лҸҷмһ‘ мӣҗлҰ¬:

flowchart TD
    A["INSERT мӢңлҸ„"] --> B{"PK/UNIQUE мӨ‘ліө?"}
    B -->|"мӨ‘ліө м—ҶмқҢ"| C["INSERT мӢӨн–ү"]
    B -->|"мӨ‘ліө мһҲмқҢ"| D["UPDATE мӢӨн–ү<br/>(ON DUPLICATE KEY UPDATE л¶Җл¶„)"]
    C --> E["'м—Ҷмңјл©ҙ л„Јкі , мһҲмңјл©ҙ к°ұмӢ н•ңлӢӨ' = MERGE/UPSERT"]
    D --> E

мқҙкұё foreachмҷҖ кІ°н•©н•ҳл©ҙ Batch UPSERTк°Җ лҗңлӢӨ:

    VALUES
    <foreach collection="studList" item="item" separator=",">
        (#{item.subjectNo}, #{item.userNo}, 'normal', 'S', 'Y')

    ON DUPLICATE KEY UPDATE
        learnerStatus = VALUES(learnerStatus),
        enrlSts = VALUES(enrlSts),
        haksaYn = VALUES(haksaYn)
</insert>

5.1 мҲҳк°•мғқ лҸҷкё°нҷ”мқҳ н•өмӢ¬ м „лһөВ¶

кІҪмғҒкөӯлҰҪлҢҖ н•ҷмӮ¬м—°лҸҷм—җм„ң мҲҳк°•мғқ мғҒнғңлҘј лҸҷкё°нҷ”н•ҳлҠ” м „лһө:

[1лӢЁкі„] м „мІҙ мҲҳк°•мғқмқ„ мқјкҙ„ "мҲҳк°•м·ЁмҶҢ" мІҳлҰ¬

  • updateSyncStudStatus

[2лӢЁкі„] APIлЎң нҷ•мқёлҗң н•ҷмғқл§Ң "м •мғҒ" ліөмӣҗ

  • mergeSyncStudBatch (foreach + ON DUPLICATE KEY)
  • learnerStatus = 'normal', enrlSts = 'S'

кІ°кіј: APIм—җ м—ҶлҠ” н•ҷмғқ = нңҙн•ҷ/мһҗнҮҙ вҶ’ мһҗлҸҷмңјлЎң м·ЁмҶҢ мғҒнғң / APIм—җ мһҲлҠ” н•ҷмғқ = мһ¬н•ҷ мӨ‘ вҶ’ м •мғҒ мғҒнғң ліөмӣҗ

мҷң мқҙ мҲңм„ңмқёк°Җ?

мқҙкұё мқҙн•ҙ лӘ» н•ҳл©ҙ м•„м§Ғ Lv1мқҙм•ј.

нҷ•мқёлҗң н•ҷмғқл§Ң м •мғҒ мІҳлҰ¬н•ҳл©ҙ м•Ҳ лҗҳлӮҳ?

м•Ҳ лҗңлӢӨ.

мқҙмң : мһҗнҮҙ/нңҙн•ҷн•ң н•ҷмғқмқ„ м–ҙл–»кІҢ м•Ңм•„?

кІҪмғҒкөӯлҰҪлҢҖ APIлҠ” "нҳ„мһ¬ мһ¬н•ҷ мӨ‘мқё н•ҷмғқ лӘ©лЎқ"л§Ң мӨҖлӢӨ. "мһҗнҮҙн•ң н•ҷмғқ лӘ©лЎқ"мқ„ л”°лЎң мЈјм§Җ м•ҠлҠ”лӢӨ.

к·ёлһҳм„ң:

  1. мқјлӢЁ м „л¶Җ м·ЁмҶҢ (к№ЁлҒ—н•ң мғҒнғң)
  2. APIм—җм„ң нҷ•мқёлҗң н•ҷмғқл§Ң ліөмӣҗ
  3. APIм—җ м—ҶлҠ” н•ҷмғқ = мһҗм—°мҠӨлҹҪкІҢ м·ЁмҶҢ мғҒнғң мң м§Җ

мқҙкІҢ "м „мІҙ лҰ¬м…Ӣ вҶ’ л¶Җл¶„ ліөмӣҗ" нҢЁн„ҙмқҙлӢӨ. Delete & Re-insert нҢЁн„ҙкіј к°ңл…җмқҙ к°ҷлӢӨ.

5.3 mergeSyncStudBatch (2лӢЁкі„: нҷ•мқёлҗң н•ҷмғқ ліөмӣҗ)В¶

6. Batch vs лӢЁкұҙ мІҳлҰ¬ м„ұлҠҘВ¶

6.1 м„ұлҠҘ л№„көҗ (1000кұҙ кё°мӨҖ)В¶

л°©мӢқ DB мҷ•ліө м»Өл„Ҙм…ҳ м җмң  мғҒлҢҖ мҶҚлҸ„
лӢЁкұҙ INSERT 1000нҡҢ кёёлӢӨ 1x (кё°мӨҖ)
Batch INSERT (foreach) 1нҡҢ м§§лӢӨ 10~50x

мҷң мқҙл ҮкІҢ м°Ёмқҙ лӮҳлғҗ?

  • лӢЁкұҙ: [Java] вҶ’ л„ӨнҠёмӣҢнҒ¬ вҶ’ [DB] вҶ’ л„ӨнҠёмӣҢнҒ¬ вҶ’ [Java] мқҙкұё 1000лІҲ л°ҳліө
  • Batch: [Java] вҶ’ л„ӨнҠёмӣҢнҒ¬ вҶ’ [DBм—җм„ң 1000кұҙ мІҳлҰ¬] вҶ’ л„ӨнҠёмӣҢнҒ¬ вҶ’ [Java] мҷ•ліө 1лІҲ

л„ӨнҠёмӣҢнҒ¬ мҷ•ліө(Round Trip)мқҙ лі‘лӘ©мқҙлӢӨ. SQL мӢӨн–ү мһҗмІҙліҙлӢӨ мҷ•ліө л№„мҡ©мқҙ нӣЁм”¬ нҒ¬лӢӨ.

6.2 DB м»Өл„Ҙм…ҳ нҡЁмңЁВ¶

м»Өл„Ҙм…ҳ н’Җ кҙҖм җ

м»Өл„Ҙм…ҳ н’Җ (ліҙнҶө 10~20к°ң) - мҙқ 10к°ңлқјкі  к°Җм •

лӢЁкұҙ мІҳлҰ¬:

  • мҠӨл Ҳл“ң Aк°Җ м»Өл„Ҙм…ҳ 1лІҲмқ„ л¬јкі  1000лІҲ мҷ•ліө
  • к·ё лҸҷм•Ҳ лӢӨлҘё мҡ”мІӯмқҖ 9к°ң м»Өл„Ҙм…ҳмңјлЎң лІ„н…Ём•ј н•Ё
  • мҳӨлһҳ кұёлҰ¬л©ҙ м»Өл„Ҙм…ҳ кі к°Ҳ вҶ’ лӢӨлҘё м„ңл№„мҠӨ мһҘм• 

Batch мІҳлҰ¬:

  • мҠӨл Ҳл“ң Aк°Җ м»Өл„Ҙм…ҳ 1лІҲмңјлЎң 1нҡҢ мҷ•ліө нӣ„ мҰүмӢң л°ҳнҷҳ
  • м»Өл„Ҙм…ҳмқ„ м§§кІҢ м“°кі  л№ЁлҰ¬ лҸҢл ӨлҶ“мқҢ
  • лӢӨлҘё мҡ”мІӯм—җ мҳҒн–Ҙ мөңмҶҢнҷ”

7. мЈјмқҳмӮ¬н•ӯВ¶

7.1 SQL л№Ҳ мӨ„ кёҲм§ҖВ¶

мқҙкұҙ MyBatisм—җм„ң мӢӨм ңлЎң нҢҢмӢұ мҳӨлҘҳлҘј мқјмңјнӮЁлӢӨ. м ҲлҢҖ кёҲм§Җ.

<!-- кёҲм§Җ: SQL мӨ‘к°„м—җ л№Ҳ мӨ„ -->
<update id="updateUser">
    UPDATE TB_USER
    SET USER_NM = #{userNm}

    WHERE USER_NO = #{userNo}
</update>

<!-- мҳ¬л°”лҰ„: л№Ҳ мӨ„ м—Ҷмқҙ м—°мҶҚ мһ‘м„ұ -->
<update id="updateUser">
    UPDATE TB_USER
    SET USER_NM = #{userNm}
    WHERE USER_NO = #{userNo}
</update>

MyBatisк°Җ XMLмқ„ нҢҢмӢұн•  л•Ң л№Ҳ мӨ„мқҙ л“Өм–ҙк°Җл©ҙ SQLмқҙ мһҳлҰ¬кұ°лӮҳ мҳҲмғҒм№ҳ лӘ»н•ң л¬ёлІ• мҳӨлҘҳк°Җ л°ңмғқн•  мҲҳ мһҲлӢӨ. нҠ№нһҲ лҸҷм Ғ SQL нғңк·ё(<if>, <where>, <foreach>) мӮ¬мқҙм—җ л№Ҳ мӨ„мқ„ л„Јмңјл©ҙ лҚ” мң„н—ҳн•ҳлӢӨ.

7.2 foreach н•ӯлӘ©мқҙ л№„м–ҙмһҲмқ„ л•Ң мІҳлҰ¬В¶

INSERT INTO TB_USER VALUES

7.3 лҢҖлҹү лҚ°мқҙн„° мӢң Batch Size мЎ°м ҲВ¶

foreachлЎң н•ң лІҲм—җ л„Ҳл¬ҙ л§ҺмқҖ лҚ°мқҙн„°лҘј л„Јмңјл©ҙ л¬ём ңк°Җ мғқкёҙлӢӨ.

Batch Size мЈјмқҳмӮ¬н•ӯ

MySQLмқҳ max_allowed_packet м ңн•ң

  • н•ҳлӮҳмқҳ SQL нҢЁнӮ· мөңлҢҖ нҒ¬кё° (кё°ліё 4MB~64MB)
  • foreachлЎң 10л§Ң кұҙ л„Јмңјл©ҙ мқҙ м ңн•ң мҙҲкіјн•  мҲҳ мһҲмқҢ

PreparedStatement нҢҢлқјлҜён„° м ңн•ң

  • MySQLмқҖ 65535к°ң м ңн•ң
  • м»¬лҹј 10к°ң * 6554кұҙ = 65540 вҶ’ мҙҲкіј!

к¶ҢмһҘ: 500~1000кұҙм”© лӮҳлҲ м„ң Batch мӢӨн–ү

мӢӨм „ мҪ”л“ң нҢЁн„ҙ:

// 1000кұҙм”© лӮҳлҲ м„ң Batch INSERT
int batchSize = 1000;
for (int i = 0; i < studList.size(); i += batchSize) {
    int end = Math.min(i + batchSize, studList.size());
    List<StudVO> subList = studList.subList(i, end);

    Map<String, Object> paramMap = new HashMap<>();
    paramMap.put("studList", subList);
    syncMapper.mergeSyncStudBatch(paramMap);
}

7.4 Batch UPDATEлҠ” foreach л°©мӢқмқҙ лӢӨлҘҙлӢӨВ¶

INSERTлҠ” VALUES (), (), () нҢЁн„ҙмқҙ мһҲм§Җл§Ң, UPDATEлҠ” к·ёлҹ° л¬ёлІ•мқҙ м—ҶлӢӨ. к°Ғк°Ғ л”°лЎң мӢӨн–үн•ҙм•ј н•ңлӢӨ.

<!-- Batch UPDATE: м—¬лҹ¬ UPDATEл¬ёмқ„ м„ёлҜёмҪңлЎ мңјлЎң м—°кІ° -->
<!-- JDBC URLм—җ allowMultiQueries=true н•„мҡ” -->
<update id="updateBatch">
    <foreach collection="list" item="item" separator=";">
        UPDATE TB_USER
        SET USER_NM = #{item.userNm}
        WHERE USER_NO = #{item.userNo}
    </foreach>
</update>

мқҙ л°©мӢқмқҖ JDBC м—°кІ° м„Өм •м—җ allowMultiQueries=trueк°Җ н•„мҡ”н•ҳлӢӨ. м—Ҷмңјл©ҙ м—¬лҹ¬ SQLмқ„ н•ң лІҲм—җ ліҙлӮҙм§Җ лӘ»н•ҙм„ң м—җлҹ¬к°Җ лӮңлӢӨ.

кІҪмғҒкөӯлҰҪлҢҖ мҪ”л“ңм—җм„ң updateSyncStudStatusлҘј Batchк°Җ м•„лӢҢ лӢЁкұҙ UPDATEлЎң мІҳлҰ¬н•ҳлҠ” мқҙмң лҸ„ мқҙкІғмқҙлӢӨ. WHERE мЎ°кұҙмңјлЎң н•ң лІҲм—җ м—¬лҹ¬ н–үмқ„ м—…лҚ°мқҙнҠён•ҳкё° л•Ңл¬ём—җ foreachк°Җ н•„мҡ” м—ҶлӢӨ.

8. нҷ•мқё л¬ём ңВ¶

л¬ём ң 1.В¶

#{} мҷҖ ${}мқҳ м°ЁмқҙлҘј м„ӨлӘ…н•ҳкі , SQL Injection кіөкІ©мқҙ к°ҖлҠҘн•ң мӘҪмқҖ м–ҙлҠҗ мӘҪмқём§Җ, к·ё мқҙмң лҠ” лӯ”м§Җ л§җн•ҙлҙҗ.

м •лӢө ліҙкё°

#{}лҠ” PreparedStatementмқҳ нҢҢлқјлҜён„° л°”мқёл”©мқ„ мӮ¬мҡ©н•ңлӢӨ. SQLмқҙ лЁјм Җ нҢҢмӢұ/м»ҙнҢҢмқјлҗң нӣ„ к°’мқҙ л°”мқёл”©лҗҳлҜҖлЎң SQL кө¬мЎ°лҘј ліҖкІҪн•  мҲҳ м—ҶлӢӨ. SQL Injection л°©м–ҙлҗЁ.

${}лҠ” л¬ёмһҗм—ҙ к·ёлҢҖлЎң м№ҳнҷҳ(String Substitution)мқҙлӢӨ. SQL нҢҢмӢұ м „м—җ к°’мқҙ л¬ёмһҗм—ҙлЎң кІ°н•©лҗҳлҜҖлЎң, м•…мқҳм Ғ мһ…л Ҙмқҙ SQL кө¬мЎ° мһҗмІҙлҘј ліҖкІҪн•  мҲҳ мһҲлӢӨ. SQL Injectionм—җ м·Ём•Ҫ.

${}к°Җ SQL Injectionм—җ м·Ём•Ҫн•ң мқҙмң лҠ” к°’мқҙ SQLмқҳ мқјл¶ҖлЎң н•ҙм„қлҗҳкё° л•Ңл¬ёмқҙлӢӨ. PreparedStatementмқҳ л°”мқёл”©мқҖ к°’мқ„ н•ӯмғҒ "лҚ°мқҙн„°"лЎңл§Ң м·Ёкёүн•ҳм§Җл§Ң, л¬ёмһҗм—ҙ м№ҳнҷҳмқҖ к°’мқ„ "SQL мҪ”л“ң"лЎң м·Ёкёүн•ңлӢӨ.

л¬ём ң 2.В¶

кІҪмғҒкөӯлҰҪлҢҖ мҲҳк°•мғқ лҸҷкё°нҷ”м—җм„ң "м „мІҙ м·ЁмҶҢ вҶ’ нҷ•мқёлҗң н•ҷмғқ ліөмӣҗ" мҲңм„ңлЎң мІҳлҰ¬н•ҳлҠ” мқҙмң лҘј м„ӨлӘ…н•ҙлҙҗ. л°ҳлҢҖлЎң "нҷ•мқёлҗң н•ҷмғқл§Ң м •мғҒ мІҳлҰ¬"н•ҳл©ҙ м•Ҳ лҗҳлҠ” мқҙмң лҠ”?

м •лӢө ліҙкё°

кІҪмғҒкөӯлҰҪлҢҖ APIлҠ” нҳ„мһ¬ мһ¬н•ҷ мӨ‘мқё н•ҷмғқ лӘ©лЎқл§Ң м ңкіөн•ҳкі , "мһҗнҮҙ/нңҙн•ҷн•ң н•ҷмғқ лӘ©лЎқ"мқҖ лі„лҸ„лЎң м ңкіөн•ҳм§Җ м•ҠлҠ”лӢӨ.
