12. REST м•„нӮӨн…ҚмІҳ¶

н•ң мӨ„ мҡ”м•Ҫ

RESTлҠ” кё°мҲ мқҙ м•„лӢҲлқј м•„нӮӨн…ҚмІҳ мҠӨнғҖмқј(м ңм•ҪмЎ°кұҙ лӘЁмқҢ)мқҙлӢӨ. 6к°Җм§Җ м ңм•Ҫмқ„ м§ҖнӮӨл©ҙ RESTful, м•Ҳ м§ҖнӮӨл©ҙ к·ёлғҘ HTTP API.

н•өмӢ¬ к°ңл…җ¶

RESTк°Җ лӯҗм•ј¶

REST = Representational State Transfer

2000л…„, Roy FieldingмқҙлқјлҠ” мӮ¬лһҢмқҙ л°•мӮ¬ л…јл¬ём—җм„ң м ңм•Ҳн•ң мӣ№ м•„нӮӨн…ҚмІҳ мҠӨнғҖмқјмқҙм•ј.

Roy Fieldingмқҙ лҲ„кө¬лғҗл©ҙ

HTTP/1.0, HTTP/1.1 лӘ…м„ёмқҳ мЈјмҡ” м Җмһҗ. Apache мӣ№ м„ңлІ„мқҳ кіөлҸҷ м°ҪмӢңмһҗ. мҰү, мӣ№мқҳ м•„лІ„м§Җ мӨ‘ н•ң лӘ…мқҙм•ј. мқҙ мӮ¬лһҢмқҙ "мӣ№мқ„ мһҳ м“°л Өл©ҙ мқҙл ҮкІҢ н•ҙлқј" н•ң кІҢ REST.

лӢЁм–ҙлҘј лңҜм–ҙліҙмһҗ:

лӢЁм–ҙ мқҳлҜё
Representational н‘ңнҳ„ (лҰ¬мҶҢмҠӨлҘј JSON, XML л“ұмңјлЎң н‘ңнҳ„)
State мғҒнғң (лҰ¬мҶҢмҠӨмқҳ нҳ„мһ¬ мғҒнғң)
Transfer м „мҶЎ (нҒҙлқјмқҙм–ёнҠёмҷҖ м„ңлІ„ к°„ мғҒнғңмқҳ н‘ңнҳ„мқ„ м „мҶЎ)

н•©м№ҳл©ҙ: лҰ¬мҶҢмҠӨмқҳ мғҒнғңлҘј н‘ңнҳ„мңјлЎң м „мҶЎн•ҳлҠ” м•„нӮӨн…ҚмІҳ мҠӨнғҖмқј.

мқҙкұ° мӨ‘мҡ”н•ҙ

RESTлҠ” н”„лЎңнҶ мҪңмқҙ м•„лӢҲм•ј. н”„л Ҳмһ„мӣҢнҒ¬лҸ„ м•„лӢҲм•ј. лқјмқҙлёҢлҹ¬лҰ¬лҸ„ м•„лӢҲм•ј. "мқҙл ҮкІҢ м„Өкі„н•ҳл©ҙ мўӢмқҖ мӣ№ мӢңмҠӨн…ңмқҙ лҗңлӢӨ"лҠ” м ңм•ҪмЎ°кұҙ лӘЁмқҢмқҙм•ј. кё°мҲ мқҙ м•„лӢҲлқј мІ н•ҷм—җ к°Җк№қлӢӨ.

1. Client-Server (нҒҙлқјмқҙм–ёнҠё-м„ңлІ„ л¶„лҰ¬)¶

нҒҙлқјмқҙм–ёнҠёмҷҖ м„ңлІ„мқҳ кҙҖмӢ¬мӮ¬лҘј л¶„лҰ¬н•ңлӢӨ.

нҒҙлқјмқҙм–ёнҠё м„ңлІ„
UI, мӮ¬мҡ©мһҗ кІҪн—ҳ лҚ°мқҙн„° м ҖмһҘ, л№„мҰҲлӢҲмҠӨ лЎңм§Ғ
"м–ҙл–»кІҢ ліҙм—¬мӨ„к№Ң" "м–ҙл–»кІҢ мІҳлҰ¬н• к№Ң"

мҷң?

  • лҸ…лҰҪм Ғ м§„нҷ”: м„ңлІ„ л°”кҝ”лҸ„ нҒҙлқјмқҙм–ёнҠё м•Ҳ кұҙл“ңл ӨлҸ„ лҗЁ. л°ҳлҢҖлҸ„ л§Ҳм°¬к°Җм§Җ.
  • нҷ•мһҘм„ұ: м„ңлІ„л§Ң л”°лЎң мҠӨмјҖмқјм—… к°ҖлҠҘ
  • мқҙмӢқм„ұ: к°ҷмқҖ APIлЎң мӣ№, м•ұ, IoT лӢӨ лҢҖмқ‘

мӢӨм „м—җм„ң

LMS м„ңлІ„(Spring)лҘј лҰ¬нҢ©нҶ л§Ғн•ҙлҸ„, API лӘ…м„ёл§Ң м•Ҳ л°”кҫёл©ҙ н”„лЎ нҠё(JSP)лҠ” мҳҒн–Ҙ м—Ҷм–ҙ. лӮҳмӨ‘м—җ JSPлҘј ReactлЎң л°”кҝ”лҸ„ к°ҷмқҖ API к·ёлҢҖлЎң м“ё мҲҳ мһҲм–ҙ.

2. Stateless (л¬ҙмғҒнғң)¶

м„ңлІ„к°Җ нҒҙлқјмқҙм–ёнҠёмқҳ мғҒнғңлҘј м ҖмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ.

лӘЁл“  мҡ”мІӯм—җлҠ” к·ё мҡ”мІӯмқ„ мІҳлҰ¬н•ҳлҠ” лҚ° н•„мҡ”н•ң лӘЁл“  м •ліҙк°Җ лӢҙкІЁ мһҲм–ҙм•ј н•ҙ.

    participant S as м„ңлІ„

    C->>S: "м•„мқҙл””лҠ” junseong" (2лӢЁкі„)
    Note right of S: мғҒнғң м ҖмһҘ: "м•„мқҙл”” л°ӣмқҢ"
    C->>S: "л№„лІҲмқҖ 1234" (3лӢЁкі„)
    Note right of S: мқҙм „ мғҒнғң м°ёмЎ°н•ҙм„ң мІҳлҰ¬

мҷң?

  • нҷ•мһҘм„ұ: м„ңлІ„к°Җ мғҒнғң м•Ҳ л“Өкі  мһҲмңјлӢҲ м•„л¬ҙ м„ңлІ„лӮҳ мҡ”мІӯ мІҳлҰ¬ к°ҖлҠҘ (лЎңл“ңл°ёлҹ°мӢұ мү¬мӣҖ)
  • мӢ лў°м„ұ: м„ңлІ„ н•ҳлӮҳ мЈҪм–ҙлҸ„ лӢӨлҘё м„ңлІ„к°Җ л°”лЎң лҢҖмІҙ
  • лӢЁмҲңм„ұ: м„ңлІ„ лЎңм§Ғмқҙ к°„лӢЁн•ҙм§җ

к·ёлҹј лЎңк·ёмқё мғҒнғңлҠ” м–ҙл–»кІҢ?

м„ңлІ„к°Җ мғҒнғңлҘј м•Ҳ к°Җм§ҖлӢҲк№Ң, нҒҙлқјмқҙм–ёнҠёк°Җ л§Ө мҡ”мІӯл§ҲлӢӨ мқёмҰқ м •ліҙлҘј ліҙлӮҙм•ј н•ҙ. мқҙкІҢ л°”лЎң нҶ нҒ°(JWT) л°©мӢқмқҙ нғ„мғқн•ң мқҙмң м•ј.

GET /api/users/me
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

л§Ө мҡ”мІӯл§ҲлӢӨ нҶ нҒ°мқ„ ліҙлӮҙм„ң "лӮҳ мқёмҰқлҗң мӮ¬мҡ©мһҗм•ј"лҘј мҰқлӘ…н•ҳлҠ” кұ°м§Җ.

3. Cacheable (мәҗмӢң к°ҖлҠҘ)¶

мқ‘лӢөм—җ мәҗмӢң к°ҖлҠҘ м—¬л¶ҖлҘј лӘ…мӢңн•ҙм•ј н•ңлӢӨ.

HTTP/1.1 200 OK
Cache-Control: max-age=3600
ETag: "abc123"

{"id": 1, "name": "кіөм§ҖмӮ¬н•ӯ лӘ©лЎқ", ...}

мҷң?

  • м„ұлҠҘ: к°ҷмқҖ лҚ°мқҙн„° лҳҗ мҡ”мІӯ м•Ҳ н•ҙлҸ„ лҗЁ вҶ’ л„ӨнҠёмӣҢнҒ¬ нҠёлһҳн”Ҫ к°җмҶҢ
  • мқ‘лӢө мҶҚлҸ„: мәҗмӢңм—җм„ң л°”лЎң л°ҳнҷҳ вҶ’ мӮ¬мҡ©мһҗ мІҙк°җ мҶҚлҸ„ н–ҘмғҒ
  • м„ңлІ„ л¶Җн•ҳ: л¶Ҳн•„мҡ”н•ң мҡ”мІӯ мӨ„м–ҙл“Ұ
мәҗмӢң м Ғн•© мәҗмӢң л¶Җм Ғн•©
кіөм§ҖмӮ¬н•ӯ лӘ©лЎқ мӢӨмӢңк°„ мұ„нҢ… л©”мӢңм§Җ
к°•мқҳ м •ліҙ нҳ„мһ¬ м ‘мҶҚмһҗ мҲҳ
мӮ¬мҡ©мһҗ н”„лЎңн•„ мқҙлҜём§Җ кІ°м ң мІҳлҰ¬ кІ°кіј
м •м Ғ лҰ¬мҶҢмҠӨ (CSS, JS) мқјнҡҢм„ұ мқёмҰқ мҪ”л“ң

4. Layered System (кі„мёөнҷ” мӢңмҠӨн…ң)¶

нҒҙлқјмқҙм–ёнҠёлҠ” м§Ғм ‘ м„ңлІ„м—җ м—°кІ°лҗҗлҠ”м§Җ, мӨ‘к°„м—җ лӯҗк°Җ мһҲлҠ”м§Җ лӘЁлҘёлӢӨ.

мҷң?

  • ліҙм•Ҳ: н”„лЎқмӢң, л°©нҷ”лІҪ к°ҷмқҖ ліҙм•Ҳ кі„мёө м¶”к°Җ к°ҖлҠҘ
  • нҷ•мһҘм„ұ: лЎңл“ңл°ёлҹ°м„ңлЎң нҠёлһҳн”Ҫ л¶„мӮ°
  • мң м—°м„ұ: CDN, мәҗмӢң м„ңлІ„ л“ұ мӨ‘к°„ кі„мёө мһҗмң лЎӯкІҢ м¶”к°Җ/м ңкұ°

мӢӨл¬ҙм—җм„ң

нҒҙлқјмқҙм–ёнҠё вҶ’ Nginx(лҰ¬лІ„мҠӨ н”„лЎқмӢң) вҶ’ Spring Boot(WAS) вҶ’ DB мқҙ кө¬мЎ°к°Җ кі„мёөнҷ” мӢңмҠӨн…ңмқҳ м „нҳ•м Ғмқё мҳҲмӢңм•ј. нҒҙлқјмқҙм–ёнҠёлҠ” Nginx л’Өм—җ лӯҗк°Җ мһҲлҠ”м§Җ м „нҳҖ лӘЁлҘҙкі , м•Ң н•„мҡ”лҸ„ м—Ҷм–ҙ.

5. Uniform Interface (к· мқјн•ң мқён„°нҺҳмқҙмҠӨ)¶

RESTмқҳ н•өмӢ¬ м°Ёлі„м җ

лӢӨлҘё 5к°ң м ңм•ҪмЎ°кұҙмқҖ лӢӨлҘё м•„нӮӨн…ҚмІҳм—җлҸ„ мһҲм–ҙ. Uniform Interfaceк°Җ RESTлҘј RESTлӢөкІҢ л§Ңл“ңлҠ” н•өмӢ¬мқҙм•ј.

4к°Җм§Җ н•ҳмң„ м ңм•ҪмңјлЎң кө¬м„ұлҸј:

5-1. лҰ¬мҶҢмҠӨ мӢқлі„ (Identification of Resources)¶

лӘЁл“  лҰ¬мҶҢмҠӨлҠ” URIлЎң кі мң н•ҳкІҢ мӢқлі„лҗңлӢӨ.

/users/123        вҶҗ 123лІҲ мӮ¬мҡ©мһҗлқјлҠ” "лҰ¬мҶҢмҠӨ"
/courses/456      вҶҗ 456лІҲ к°•мўҢлқјлҠ” "лҰ¬мҶҢмҠӨ"
/lectures/789     вҶҗ 789лІҲ к°•мқҳлқјлҠ” "лҰ¬мҶҢмҠӨ"

5-2. н‘ңнҳ„мқ„ нҶөн•ң лҰ¬мҶҢмҠӨ мЎ°мһ‘ (Manipulation through Representations)¶

лҰ¬мҶҢмҠӨлҘј м§Ғм ‘ мЈјкі л°ӣлҠ” кІҢ м•„лӢҲлқј, лҰ¬мҶҢмҠӨмқҳ н‘ңнҳ„(representation)мқ„ мЈјкі л°ӣм•„.

DBм—җ м ҖмһҘлҗң мӮ¬мҡ©мһҗ лҚ°мқҙн„° (лҰ¬мҶҢмҠӨ к·ё мһҗмІҙ)
    вҶ“ н‘ңнҳ„мңјлЎң ліҖнҷҳ
JSON нҳ•нғң (лҰ¬мҶҢмҠӨмқҳ н‘ңнҳ„)
    вҶ“ м „мҶЎ
нҒҙлқјмқҙм–ёнҠёк°Җ л°ӣмқҢ

к°ҷмқҖ лҰ¬мҶҢмҠӨлқјлҸ„ JSONмңјлЎң мӨ„ мҲҳлҸ„ мһҲкі , XMLлЎң мӨ„ мҲҳлҸ„ мһҲм–ҙ. н‘ңнҳ„мқҙ лӢӨлҘј лҝҗ лҰ¬мҶҢмҠӨлҠ” к°ҷм•„.

5-3. мһҗкё° м„ңмҲ м Ғ л©”мӢңм§Җ (Self-descriptive Messages)¶

л©”мӢңм§Җл§Ң ліҙл©ҙ лӯҳ н•ҙм•ј н•ҳлҠ”м§Җ лӢӨ м•Ң мҲҳ мһҲм–ҙм•ј н•ҙ.

GET /users/123 HTTP/1.1
Host: api.example.com
Accept: application/json
Authorization: Bearer eyJhbG...

мқҙ мҡ”мІӯл§Ң ліҙл©ҙ: "api.example.comмқҳ 123лІҲ мӮ¬мҡ©мһҗлҘј JSONмңјлЎң лӢ¬лқј, мқёмҰқ нҶ нҒ°мқҖ мқҙкұ°" -- лӢӨ м•Ң мҲҳ мһҲм§Җ?

5-4. HATEOAS (Hypermedia As The Engine Of Application State)¶

мқ‘лӢөм—җ лӢӨмқҢм—җ н•  мҲҳ мһҲлҠ” н–үлҸҷ(л§ҒнҒ¬)мқҙ нҸ¬н•ЁлҸјм•ј н•ҙ.

{
  "id": 123,
  "name": "нҷҚкёёлҸҷ",
  "_links": {
    "self": { "href": "/users/123" },
    "courses": { "href": "/users/123/courses" },
    "update": { "href": "/users/123", "method": "PUT" },
    "delete": { "href": "/users/123", "method": "DELETE" }
  }
}

нҳ„мӢӨмқҖ...

HATEOASлҘј м ңлҢҖлЎң кө¬нҳ„н•ҳлҠ” APIлҠ” кұ°мқҳ м—Ҷм–ҙ. мқҙлЎ м ҒмңјлЎңлҠ” мӨ‘мҡ”н•ҳм§Җл§Ң, мӢӨл¬ҙм—җм„ңлҠ” лҢҖл¶Җл¶„ API л¬ём„ңлЎң лҢҖмІҙн•ҙ. "мқҙмғҒм Ғмқё REST"мҷҖ "нҳ„мӢӨмқҳ REST"лҠ” м°Ёмқҙк°Җ мһҲлӢӨлҠ” кұ°м•ј.

6. Code on Demand (м„ нғқ мӮ¬н•ӯ)¶

м„ңлІ„к°Җ нҒҙлқјмқҙм–ёнҠём—җ мӢӨн–ү к°ҖлҠҘн•ң мҪ”л“ңлҘј м „мҶЎн•  мҲҳ мһҲлӢӨ.

мҳҲлҘј л“Өм–ҙ, м„ңлІ„к°Җ JavaScript мҪ”л“ңлҘј ліҙлӮҙм„ң нҒҙлқјмқҙм–ёнҠё(лёҢлқјмҡ°м Җ)к°Җ мӢӨн–үн•ҳлҠ” кІғ.

мқҙкұҙ м„ нғқмқҙм•ј

6к°Җм§Җ м ңм•Ҫ мӨ‘ мң мқјн•ҳкІҢ м„ нғқ мӮ¬н•ӯ(optional)мқҙм•ј. лӮҳлЁём§Җ 5к°ңлҠ” л°ҳл“ңмӢң м§Җмјңм•ј RESTfulмқҙлқјкі  н•  мҲҳ мһҲм–ҙ.

"REST API" vs "HTTP API"¶

л¶ҲнҺён•ң м§„мӢӨ

м„ёмғҒм—җ мһҲлҠ” лҢҖл¶Җл¶„мқҳ "REST API"лҠ” м§„м§ң RESTfulн•ҳм§Җ м•Ҡм•„. к·ёлғҘ HTTPлЎң JSON мЈјкі л°ӣлҠ” APIлҘј REST APIлқјкі  л¶ҖлҘҙлҠ” кІҪмҡ°к°Җ лҢҖл¶Җл¶„мқҙм•ј.

нҳ„мӢӨм Ғмқё лӘ©н‘ң

Level 2лҘј мһҳ м§ҖнӮӨлҠ” кІғмқҙ нҳ„мӢӨм Ғмқё лӘ©н‘ңм•ј. Level 3(HATEOAS)к№Ңм§Җ мҷ„лІҪн•ҳкІҢ кө¬нҳ„н•ҳлҠ” кұҙ мҳӨлІ„м—”м§ҖлӢҲм–ҙл§Ғмқё кІҪмҡ°к°Җ л§Һм•„.

к·ёлһҳм„ң м—„л°ҖнһҲ л§җн•ҳл©ҙ "REST API"ліҙлӢӨ "HTTP API" лҳҗлҠ” "RESTish API"к°Җ м •нҷ•н•ң н‘ңнҳ„мқҙм•ј. н•ҳм§Җл§Ң м—…кі„ кҙҖн–үмғҒ Level 2л§Ң м§ҖмјңлҸ„ "REST API"лқјкі  л¶ҖлҘҙлҠ” кІҪмҡ°к°Җ лҢҖл¶Җл¶„мқҙм•ј.

мҷң RESTк°Җ мқҙкІјлӮҳ¶

2000л…„лҢҖ мҙҲмӨ‘л°ҳ, Web APIмқҳ лҢҖн‘ң кё°мҲ мқҖ SOAPмқҙм—Ҳм–ҙ.

н•ӯлӘ© SOAP REST
н”„лЎңнҶ мҪң лҸ…мһҗ н”„лЎңнҶ мҪң (XML кё°л°ҳ) HTTP к·ёлҢҖлЎң нҷңмҡ©
л©”мӢңм§Җ нҳ•мӢқ XMLл§Ң к°ҖлҠҘ JSON, XML л“ұ мһҗмң 
ліөмһЎлҸ„ WSDL, мҠӨнӮӨл§Ҳ, м—”лІЁлЎңн”„... URL + л©”м„ңл“ң + JSON
н•ҷмҠө кіЎм„  к°ҖнҢҢлҰ„ мҷ„л§Ңн•Ё
н‘ңмӨҖ м—„кІ©н•ң н‘ңмӨҖ к°Җмқҙл“ңлқјмқё мҲҳмӨҖ
лҸ„кө¬ мқҳмЎҙм„ұ м „мҡ© лҸ„кө¬ н•„мҡ” лёҢлқјмҡ°м Җ, curlлЎң к°ҖлҠҘ

RESTк°Җ мқҙкёҙ мқҙмң :

  1. лӢЁмҲңн•Ё: HTTPлҘј к·ёлҢҖлЎң м“°лӢҲк№Ң л°°мҡ°кё° мү¬мӣҢ
  2. к°ҖлІјмӣҖ: JSONмқҖ XMLліҙлӢӨ к°ҖліҚкі  мқҪкё° мү¬мӣҢ
  3. лҸ„кө¬ лҸ…лҰҪ: лі„лҸ„ лҸ„кө¬ м—Ҷмқҙ HTTPл§Ң ліҙлӮҙл©ҙ лҸј
  4. мӣ№ м№ңнҷ”м Ғ: мӣ№мқҳ кё°мЎҙ мқён”„лқј(мәҗмӢң, н”„лЎқмӢң)лҘј к·ёлҢҖлЎң нҷңмҡ©

мЈјмқҳмӮ¬н•ӯ / н•Ём •¶

нқ”н•ң м°©к°Ғ

м°©к°Ғ 1: "JSON м“°л©ҙ RESTлӢӨ"

м•„лӢҲм•ј. JSONмқҖ лҚ°мқҙн„° нҳ•мӢқмқј лҝҗмқҙм•ј. REST м ңм•Ҫмқ„ м§Җмјңм•ј RESTм•ј. SOAPлҸ„ JSON м“ё мҲҳ мһҲкі , RESTлҸ„ XML м“ё мҲҳ мһҲм–ҙ.

м°©к°Ғ 2: "RESTлҠ” HTTPм—җм„ңл§Ң лҸҷмһ‘н•ңлӢӨ"

мқҙлЎ м ҒмңјлЎңлҠ” лӢӨлҘё н”„лЎңнҶ мҪңм—җм„ңлҸ„ к°ҖлҠҘн•ҙ. н•ҳм§Җл§Ң мӢӨм§Ҳм ҒмңјлЎң HTTPмҷҖ н•Ёк»ҳ м“°мқҙлҠ” кІҢ 99.9%м•ј.

м°©к°Ғ 3: "Statelessл©ҙ лЎңк·ёмқё кё°лҠҘ лӘ» л§Ңл“ лӢӨ"

StatelessлҠ” м„ңлІ„к°Җ м„ём…ҳ мғҒнғңлҘј м•Ҳ к°Җм§„лӢӨлҠ” кұ°м•ј. нҶ нҒ°(JWT)мқ„ нҒҙлқјмқҙм–ёнҠёк°Җ л§ӨлІҲ ліҙлӮҙл©ҙ лЎңк·ёмқё мң м§Җ к°ҖлҠҘн•ҙ. мғҒнғңлҘј нҒҙлқјмқҙм–ёнҠёк°Җ л“Өкі  лӢӨлӢҲлҠ” кұ°м§Җ.

л©ҙм ‘м—җм„ң мһҗмЈј лӮҳмҳӨлҠ” м§Ҳл¬ё

"REST APIмқҳ нҠ№м§•мқ„ м„ӨлӘ…н•ҳм„ёмҡ”" вҶ’ 6к°Җм§Җ м ңм•ҪмЎ°кұҙмқ„ л§җн•  мҲҳ мһҲм–ҙм•ј н•ҙ. "RESTfulн•ҳлӢӨлҠ” кІҢ лӯ” лң»мқҙм—җмҡ”?" вҶ’ 6к°Җм§Җ м ңм•Ҫмқ„ м§ҖнӮЁлӢӨлҠ” лң». "RESTмҷҖ SOAPмқҳ м°Ёмқҙ?" вҶ’ мң„ л№„көҗн‘ң лӮҙмҡ©.

м •лҰ¬¶

м ңм•ҪмЎ°кұҙ н•өмӢ¬ мҷң н•„мҡ”н•ҙ
Client-Server кҙҖмӢ¬мӮ¬ л¶„лҰ¬ лҸ…лҰҪм Ғ м§„нҷ”
Stateless м„ңлІ„м—җ мғҒнғң м—ҶмқҢ нҷ•мһҘм„ұ, мӢ лў°м„ұ
Cacheable мәҗмӢң к°ҖлҠҘ лӘ…мӢң м„ұлҠҘ н–ҘмғҒ
Layered System мӨ‘к°„ кі„мёө н—Ҳмҡ© ліҙм•Ҳ, нҷ•мһҘм„ұ
Uniform Interface мқјкҙҖлҗң мқён„°нҺҳмқҙмҠӨ RESTмқҳ н•өмӢ¬ м°Ёлі„м җ
Code on Demand мҪ”л“ң м „мҶЎ (м„ нғқ) нҒҙлқјмқҙм–ёнҠё нҷ•мһҘм„ұ

кё°м–өлІ•

C-S-C-L-U-C (м”Ё-м—җмҠӨ-м”Ё-м—ҳ-мң -м”Ё) Client-Server / Stateless / Cacheable / Layered / Uniform / Code on Demand

нҷ•мқё л¬ём ң¶

л¬ём ң 1

RESTлҠ” н”„лЎңнҶ мҪңмқёк°Җ, м•„нӮӨн…ҚмІҳ мҠӨнғҖмқјмқёк°Җ? м°Ёмқҙк°Җ лӯ”м§Җ м„ӨлӘ…н•ҙлҙҗ.

м •лӢө ліҙкё°

RESTлҠ” м•„нӮӨн…ҚмІҳ мҠӨнғҖмқјмқҙм•ј.

  • н”„лЎңнҶ мҪң: м—„кІ©н•ң к·ңм№ҷ. м§ҖнӮӨм§Җ м•Ҡмңјл©ҙ нҶөмӢ  л¶Ҳк°Җ. (мҳҲ: HTTP, TCP)
  • м•„нӮӨн…ҚмІҳ мҠӨнғҖмқј: м„Өкі„ к°Җмқҙл“ңлқјмқё/м ңм•ҪмЎ°кұҙ лӘЁмқҢ. м•Ҳ м§ҖмјңлҸ„ лҸҷмһ‘мқҖ н•ҙ. лӢӨл§Ң мўӢмқҖ м„Өкі„к°Җ м•„лӢҗ лҝҗ.

RESTлҠ” "мқҙл ҮкІҢ н•ҳл©ҙ мўӢмқҖ мӣ№ мӢңмҠӨн…ңмқҙ лҗңлӢӨ"лҠ” м ңм•ҪмЎ°кұҙ лӘЁмқҢмқҙм§Җ, м•Ҳ м§ҖнӮӨл©ҙ нҶөмӢ мқҙ м•Ҳ лҗҳлҠ” кұҙ м•„лӢҲм•ј.

л¬ём ң 2

Stateless м ңм•Ҫмқҳ мқҳлҜёмҷҖ, мқҙкІҢ лЎңк·ёмқё кё°лҠҘкіј м–ҙл–»кІҢ м–‘лҰҪн•ҳлҠ”м§Җ м„ӨлӘ…н•ҙлҙҗ.

м •лӢө ліҙкё°

Stateless: м„ңлІ„к°Җ нҒҙлқјмқҙм–ёнҠёмқҳ мғҒнғң(м„ём…ҳ)лҘј м ҖмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ. лӘЁл“  мҡ”мІӯм—җ мІҳлҰ¬м—җ н•„мҡ”н•ң лӘЁл“  м •ліҙк°Җ нҸ¬н•Ёлҗҳм–ҙм•ј н•ңлӢӨ.

лЎңк·ёмқёкіјмқҳ м–‘лҰҪ: м„ңлІ„к°Җ м„ём…ҳмқ„ м ҖмһҘн•ҳлҠ” лҢҖмӢ , нҒҙлқјмқҙм–ёнҠёк°Җ мқёмҰқ нҶ нҒ°(JWT л“ұ)мқ„ л§Ө мҡ”мІӯл§ҲлӢӨ н—ӨлҚ”м—җ лӢҙм•„ ліҙлӮёлӢӨ. м„ңлІ„лҠ” нҶ нҒ°мқ„ кІҖмҰқл§Ң н•ҳкі , лі„лҸ„мқҳ м„ём…ҳ мғҒнғңлҘј к°Җм§Җм§Җ м•Ҡм•„. мғҒнғңлҘј м„ңлІ„к°Җ м•„лӢҢ нҒҙлқјмқҙм–ёнҠёк°Җ кҙҖлҰ¬н•ҳлҠ” кІғ.

л¬ём ң 3

Uniform Interfaceмқҳ 4к°Җм§Җ н•ҳмң„ м ңм•Ҫмқ„ лӮҳм—ҙн•ҳкі , нҳ„мӢӨм—җм„ң к°ҖмһҘ мһҳ м•Ҳ м§Җмјңм§ҖлҠ” кұҙ лӯ”м§Җ л§җн•ҙлҙҗ.

м •лӢө ліҙкё°
  1. лҰ¬мҶҢмҠӨ мӢқлі„ (URIлЎң лҰ¬мҶҢмҠӨ кі мң  мӢқлі„)
  2. н‘ңнҳ„мқ„ нҶөн•ң лҰ¬мҶҢмҠӨ мЎ°мһ‘ (лҰ¬мҶҢмҠӨмқҳ н‘ңнҳ„(JSON л“ұ)мқ„ мЈјкі л°ӣмқҢ)
  3. мһҗкё° м„ңмҲ м Ғ л©”мӢңм§Җ (л©”мӢңм§Җл§Ң ліҙл©ҙ лӯҳ н•ҙм•ј н•ҳлҠ”м§Җ м•Ң мҲҳ мһҲмқҢ)
  4. HATEOAS (мқ‘лӢөм—җ лӢӨмқҢ н–үлҸҷ л§ҒнҒ¬ нҸ¬н•Ё)

к°ҖмһҘ м•Ҳ м§Җмјңм§ҖлҠ” кұҙ HATEOAS. лҢҖл¶Җл¶„мқҳ APIк°Җ мқ‘лӢөм—җ н•ҳмқҙнҚјлҜёл””м–ҙ л§ҒнҒ¬лҘј нҸ¬н•Ён•ҳм§Җ м•Ҡм•„. API л¬ём„ңлЎң лҢҖмІҙн•ҳлҠ” кІҢ нҳ„мӢӨмқҙм•ј.

л¬ём ң 4

Richardson Maturity Modelм—җм„ң Level 2мҷҖ Level 3мқҳ м°Ёмқҙк°Җ лӯҗм•ј? лҢҖл¶Җл¶„мқҳ "REST API"лҠ” м–ҙлҠҗ л ҲлІЁмқҙм•ј?

м •лӢө ліҙкё°
  • Level 2: HTTP л©”м„ңл“ңлҘј мҳ¬л°”лҘҙкІҢ нҷңмҡ© (GET=мЎ°нҡҢ, POST=мғқм„ұ, PUT=мҲҳм •, DELETE=мӮӯм ң)
  • Level 3: Level 2 + HATEOAS кө¬нҳ„ (мқ‘лӢөм—җ н•ҳмқҙнҚјлҜёл””м–ҙ л§ҒнҒ¬ нҸ¬н•Ё)

лҢҖл¶Җл¶„мқҳ "REST API"лҠ” Level 2м—җ н•ҙлӢ№н•ҙ. Level 3к№Ңм§Җ кө¬нҳ„н•ҳлҠ” кІҪмҡ°лҠ” к·№нһҲ л“ңл¬јм–ҙ. к·ёлһҳм„ң м—„л°ҖнһҲ л§җн•ҳл©ҙ "RESTful API"к°Җ м•„лӢҲлқј "HTTP API"м—җ к°Җк№ҢмӣҢ.
